Homework: Build a Task Management API System

Create a complete REST API with authentication and JavaScript frontend

Difficulty: Advanced Estimated Time: 8-10 hours

Project Overview

You will build a comprehensive task management system that demonstrates mastery of WordPress REST API development. This system will include custom endpoints, multiple authentication methods, real-time updates, and a modern JavaScript frontend application.

Learning Goals

  • Design and implement custom REST API endpoints
  • Implement multiple authentication methods
  • Build a complete JavaScript application
  • Handle real-time updates
  • Create data export functionality
  • Implement comprehensive error handling

Project Requirements

Part 1: Backend API Development

Custom Post Types

  • Task CPT:
    • Title, description, status (todo, in-progress, completed)
    • Priority (low, medium, high, urgent)
    • Due date, estimated hours, actual hours
    • Assignee (user relationship)
    • Project relationship
    • Tags and categories
    • Attachments support
  • Project CPT:
    • Name, description, status
    • Start date, end date
    • Team members (user relationships)
    • Progress percentage
    • Budget, spent amount
  • Time Entry CPT:
    • Task relationship
    • User, date, hours
    • Description of work
    • Billable flag

REST API Endpoints

  • Authentication:
    • POST /taskapi/v1/auth/login (JWT authentication)
    • POST /taskapi/v1/auth/refresh
    • POST /taskapi/v1/auth/logout
    • GET /taskapi/v1/auth/validate
  • Tasks:
    • GET /taskapi/v1/tasks (with filters)
    • GET /taskapi/v1/tasks/{id}
    • POST /taskapi/v1/tasks
    • PUT /taskapi/v1/tasks/{id}
    • DELETE /taskapi/v1/tasks/{id}
    • POST /taskapi/v1/tasks/{id}/complete
    • POST /taskapi/v1/tasks/{id}/assign
    • GET /taskapi/v1/tasks/{id}/time-entries
  • Projects:
    • GET /taskapi/v1/projects
    • GET /taskapi/v1/projects/{id}
    • GET /taskapi/v1/projects/{id}/tasks
    • GET /taskapi/v1/projects/{id}/team
    • POST /taskapi/v1/projects/{id}/add-member
  • Reports:
    • GET /taskapi/v1/reports/dashboard
    • GET /taskapi/v1/reports/time-tracking
    • GET /taskapi/v1/reports/productivity
    • GET /taskapi/v1/reports/export (CSV/PDF)
  • Real-time:
    • GET /taskapi/v1/stream (Server-sent events)
    • WebSocket endpoint for live updates

Part 2: Authentication Implementation

  • JWT token authentication with refresh tokens
  • Application passwords support
  • Cookie authentication for logged-in users
  • API key authentication for external services
  • Rate limiting per authentication method
  • Token blacklist for logout

Part 3: Frontend Application

Dashboard Features

  • Task list with filtering and sorting
  • Kanban board view (drag and drop)
  • Calendar view with due dates
  • Quick task creation
  • Real-time updates
  • Search functionality

Task Management

  • Create/Edit task modal
  • Task details view
  • Comments system
  • File attachments
  • Time tracking widget
  • Task assignment

Project Management

  • Project overview
  • Team members management
  • Progress visualization
  • Budget tracking
  • Gantt chart view

Reports

  • Time tracking reports
  • Productivity metrics
  • Charts and graphs
  • Export functionality

Main Plugin File


<?php
/**
 * Plugin Name: Task Management API
 * Description: Complete task management system with REST API
 * Version: 1.0.0
 * Author: Your Name
 */

// Prevent direct access
if (!defined('ABSPATH')) {
    exit;
}

// Define constants
define('TMA_VERSION', '1.0.0');
define('TMA_PLUGIN_DIR', plugin_dir_path(__FILE__));
define('TMA_PLUGIN_URL', plugin_dir_url(__FILE__));

// Main plugin class
class Task_Management_API {
    
    private static $instance = null;
    
    public static function get_instance() {
        if (null === self::$instance) {
            self::$instance = new self();
        }
        return self::$instance;
    }
    
    private function __construct() {
        $this->load_dependencies();
        $this->init_hooks();
    }
    
    private function load_dependencies() {
        require_once TMA_PLUGIN_DIR . 'includes/class-post-types.php';
        require_once TMA_PLUGIN_DIR . 'includes/class-rest-api.php';
        require_once TMA_PLUGIN_DIR . 'includes/class-authentication.php';
        require_once TMA_PLUGIN_DIR . 'includes/class-jwt-auth.php';
        require_once TMA_PLUGIN_DIR . 'includes/class-reports.php';
        require_once TMA_PLUGIN_DIR . 'includes/class-realtime.php';
    }
    
    private function init_hooks() {
        add_action('init', [$this, 'init']);
        add_action('rest_api_init', [$this, 'rest_api_init']);
        add_action('wp_enqueue_scripts', [$this, 'enqueue_scripts']);
        
        register_activation_hook(__FILE__, [$this, 'activate']);
        register_deactivation_hook(__FILE__, [$this, 'deactivate']);
    }
    
    public function init() {
        new TMA_Post_Types();
    }
    
    public function rest_api_init() {
        new TMA_REST_API();
        new TMA_Authentication();
        new TMA_JWT_Auth();
        new TMA_Reports();
        new TMA_Realtime();
    }
    
    public function enqueue_scripts() {
        if (is_page('task-dashboard')) {
            wp_enqueue_script(
                'tma-app',
                TMA_PLUGIN_URL . 'frontend/dist/app.js',
                [],
                TMA_VERSION,
                true
            );
            
            wp_localize_script('tma-app', 'tmaConfig', [
                'apiUrl' => rest_url('taskapi/v1'),
                'nonce' => wp_create_nonce('wp_rest'),
            ]);
        }
    }
    
    public function activate() {
        // Create database tables
        $this->create_tables();
        
        // Create dashboard page
        $this->create_dashboard_page();
        
        flush_rewrite_rules();
    }
    
    public function deactivate() {
        flush_rewrite_rules();
    }
    
    private function create_tables() {
        global $wpdb;
        
        $charset_collate = $wpdb->get_charset_collate();
        
        // JWT tokens table
        $table_name = $wpdb->prefix . 'tma_tokens';
        $sql = "CREATE TABLE IF NOT EXISTS $table_name (
            id bigint(20) UNSIGNED NOT NULL AUTO_INCREMENT,
            user_id bigint(20) UNSIGNED NOT NULL,
            token_hash varchar(255) NOT NULL,
            refresh_token_hash varchar(255) NOT NULL,
            expires_at datetime NOT NULL,
            created_at datetime DEFAULT CURRENT_TIMESTAMP,
            PRIMARY KEY (id),
            KEY user_id (user_id),
            KEY token_hash (token_hash)
        ) $charset_collate;";
        
        require_once(ABSPATH . 'wp-admin/includes/upgrade.php');
        dbDelta($sql);
    }
    
    private function create_dashboard_page() {
        $page = get_page_by_path('task-dashboard');
        
        if (!$page) {
            wp_insert_post([
                'post_title' => 'Task Dashboard',
                'post_name' => 'task-dashboard',
                'post_content' => '
',
                'post_status' => 'publish',
                'post_type' => 'page',
            ]);
        }
    }
}

// Initialize plugin
Task_Management_API::get_instance();

Pro Tips

  • Start with the API: Build and test your backend before starting the frontend
  • Use Postman: Test all endpoints thoroughly before implementing in JavaScript
  • Implement auth early: Get authentication working before building features
  • Use version control: Commit frequently with meaningful messages
  • Handle errors gracefully: Always provide meaningful error messages
  • Optimize queries: Use proper indexes and avoid N+1 problems
  • Cache strategically: Cache expensive operations but invalidate properly
  • Document as you go: Write documentation while building, not after

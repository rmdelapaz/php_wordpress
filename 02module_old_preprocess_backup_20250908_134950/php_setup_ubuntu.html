Introduction to Development Environments on Ubuntu

Welcome to our first session on PHP development environment setup, specifically tailored for Ubuntu Linux! Before we dive into writing PHP code, we need to establish a proper development environment on your Ubuntu system. Think of this as preparing your workshop—a chef needs a well-organized kitchen with quality tools readily available, and similarly, a PHP developer needs a properly configured environment where all components work together seamlessly.

A proper Ubuntu PHP development environment allows you to:

  • Write, test, and debug PHP code efficiently
  • Simulate a production server locally
  • Develop without an internet connection
  • Experiment safely without affecting live websites
  • Speed up the development cycle
  • Take advantage of Linux's server capabilities

Ubuntu is an excellent choice for PHP development because many production servers run on Linux distributions. By developing on Ubuntu, you're working in an environment very similar to where your code will eventually be deployed.

Setting Up XAMPP on Ubuntu

XAMPP (Cross-Platform, Apache, MySQL, PHP, Perl) is one of the most popular all-in-one packages for setting up a PHP development environment. It's designed to be easy to install and use, making it ideal for beginners.

Step 1: Download XAMPP

First, we need to download the XAMPP installer for Linux:

  1. Visit the official XAMPP website at https://www.apachefriends.org/download.html
  2. Scroll down to the Linux section and select the version with your preferred PHP version (e.g., 8.2.x)
  3. Download the installer (.run file)

Step 2: Make the Installer Executable

Open your terminal and navigate to the directory where you downloaded the installer:

cd ~/Downloads
chmod +x xampp-linux-*-installer.run

This changes the file permissions to make it executable.

Step 3: Run the Installer

Now run the installer with administrator privileges:

sudo ./xampp-linux-*-installer.run

A graphical installer will appear. Follow these steps:

  1. Click "Next" to begin the installation
  2. On the component selection screen, keep the default selections (or customize as needed)
  3. Click "Next" to confirm the installation location (/opt/lampp by default)
  4. Click "Next" to start the installation process
  5. Wait for the installation to complete
  6. When finished, you can choose to launch the XAMPP control panel immediately

Step 4: Starting XAMPP

You can start XAMPP in two ways:

Using the Control Panel

sudo /opt/lampp/manager-linux-x64.run

This launches the graphical XAMPP control panel where you can start/stop services.

Using the Command Line

sudo /opt/lampp/lampp start

This starts all XAMPP services (Apache, MySQL, etc.) at once. You can also use:

  • sudo /opt/lampp/lampp stop - Stop all services
  • sudo /opt/lampp/lampp restart - Restart all services
  • sudo /opt/lampp/lampp status - Check the status of services

Step 5: Verifying the Installation

Open your web browser and navigate to http://localhost or http://127.0.0.1. You should see the XAMPP welcome page.

To check if PHP is working correctly, create a test file:

  1. Create a file at /opt/lampp/htdocs/info.php with the following content:
<?php
phpinfo();
?>
  • Navigate to http://localhost/info.php in your browser
  • You should see a detailed page with PHP configuration information

    • Creating a Desktop Shortcut (Optional)

    For easier access, you can create a desktop shortcut:

    sudo nano ~/.local/share/applications/xampp-control-panel.desktop

    Add the following content:

    [Desktop Entry]

    Save and close the file (Ctrl+O, Enter, Ctrl+X). You'll need to make it executable:

    chmod +x ~/.local/share/applications/xampp-control-panel.desktop

    Security Considerations

    By default, XAMPP is configured for convenience, not security. For a more secure setup:

    sudo /opt/lampp/lampp security

    This will launch a security wizard that helps you set passwords for MySQL, phpMyAdmin, and other components.

    Installing Individual Components (LAMP Stack)

    For a more Ubuntu-native approach, you can install each component individually using the package manager. This gives you more control and better integration with the system.

    Step 1: Update System Packages

    sudo apt update
sudo apt upgrade

    Step 2: Install Apache Web Server

    sudo apt install apache2

    After installation, Apache should start automatically. Verify it's running:

    sudo systemctl status apache2

    You can also check by opening http://localhost in your browser, which should display the Apache default page.

    Step 3: Install MySQL (or MariaDB)

    You can install either MySQL or MariaDB (a community-developed fork of MySQL):

    Option 1: MySQL

    sudo apt install mysql-server

    Secure your MySQL installation:

    sudo mysql_secure_installation

    Option 2: MariaDB

    sudo apt install mariadb-server

    Secure your MariaDB installation:

    sudo mysql_secure_installation

    Step 4: Install PHP

    sudo apt install php libapache2-mod-php php-mysql

    This installs PHP, the Apache PHP module, and the PHP MySQL extension.

    Step 5: Install Additional PHP Extensions

    Depending on your needs, you might want to install additional PHP extensions:

    sudo apt install php-curl php-json php-cgi php-gd php-mbstring php-xml php-xmlrpc php-zip

    Step 6: Install phpMyAdmin (Optional)

    phpMyAdmin provides a web interface to manage your MySQL databases:

    sudo apt install phpmyadmin

    During installation:

    1. Select Apache2 when prompted for the web server
    2. Choose Yes when asked to configure a database for phpMyAdmin
    3. Enter a password for the phpMyAdmin application

    Step 7: Configure Apache to Prioritize PHP Files

    Edit the Apache configuration file:

    sudo nano /etc/apache2/mods-enabled/dir.conf

    Move index.php to be first in the DirectoryIndex list:

    <IfModule mod_dir.c>
    DirectoryIndex index.php index.html index.cgi index.pl index.xhtml index.htm
</IfModule>

    Step 8: Restart Apache to Apply Changes

    sudo systemctl restart apache2

    Step 9: Verify PHP Installation

    Create a test PHP file:

    echo '<?php phpinfo(); ?>' | sudo tee /var/www/html/info.php

    Open http://localhost/info.php in your browser to see the PHP information page.

    Managing Services

    Ubuntu uses systemd to manage services. Here are some common commands:

    # Apache
sudo systemctl start apache2    # Start Apache
sudo systemctl stop apache2     # Stop Apache
sudo systemctl restart apache2  # Restart Apache
sudo systemctl status apache2   # Check Apache status

# MySQL/MariaDB
sudo systemctl start mysql      # Start MySQL
sudo systemctl stop mysql       # Stop MySQL
sudo systemctl restart mysql    # Restart MySQL
sudo systemctl status mysql     # Check MySQL status

    Setting Up Development Environment with Docker

    Docker provides a modern, containerized approach to development environments. It offers the best isolation and consistency between development and production.

    flowchart LR A[Docker Engine] --> B[PHP Container] A --> C[MySQL Container] A --> D[phpMyAdmin Container] B <--> C C <--> D B --> E[Your Code Volume] style A fill:#f96,stroke:#333,stroke-width:2px style B fill:#6c9,stroke:#333,stroke-width:2px style C fill:#69f,stroke:#333,stroke-width:2px style D fill:#fc6,stroke:#333,stroke-width:2px style E fill:#ccc,stroke:#333,stroke-width:2px

    Step 1: Install Docker on Ubuntu

    # Update package index
sudo apt update

# Install prerequisites
sudo apt install apt-transport-https ca-certificates curl software-properties-common

# Add Docker's official GPG key
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# Add the Docker repository
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# Update package index again
sudo apt update

# Install Docker Engine
sudo apt install docker-ce docker-ce-cli containerd.io

# Install Docker Compose
sudo apt install docker-compose-plugin

    Step 2: Add Your User to the Docker Group

    This allows you to run Docker commands without sudo:

    sudo usermod -aG docker $USER

    Log out and back in for the changes to take effect, or run:

    newgrp docker

    Step 3: Verify Docker Installation

    docker --version
docker compose version
docker run hello-world

    Step 4: Create a Project Directory

    mkdir -p ~/php-projects/my-first-project
cd ~/php-projects/my-first-project

    Step 5: Create Docker Compose Configuration

    Create a docker-compose.yml file:

    nano docker-compose.yml

    Add the following content:

    version: '3.8'

services:
  # PHP with Apache service
  php:
    image: php:8.2-apache
    container_name: php_dev
    ports:
      - "80:80"
    volumes:
      - ./src:/var/www/html
    depends_on:
      - mysql
    restart: always
    environment:
      - MYSQL_HOST=mysql
      - MYSQL_USER=phpuser
      - MYSQL_PASSWORD=phppassword
      - MYSQL_DB=phpdb

  # MySQL service
  mysql:
    image: mysql:8.0
    container_name: mysql_dev
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    restart: always
    environment:
      - MYSQL_ROOT_PASSWORD=rootpassword
      - MYSQL_DATABASE=phpdb
      - MYSQL_USER=phpuser
      - MYSQL_PASSWORD=phppassword

  # phpMyAdmin service
  phpmyadmin:
    image: phpmyadmin/phpmyadmin
    container_name: phpmyadmin_dev
    ports:
      - "8080:80"
    depends_on:
      - mysql
    restart: always
    environment:
      - PMA_HOST=mysql
      - PMA_PORT=3306
      - MYSQL_ROOT_PASSWORD=rootpassword

volumes:
  mysql_data:
    driver: local

    Step 6: Create PHP Source Directory

    mkdir -p src

    Step 7: Create a Test PHP File

    nano src/index.php

    Add some sample PHP code:

    <!DOCTYPE html>
<html>
<head>
    <title>PHP Docker Test</title>
</head>
<body>
    <h1>PHP Docker Test</h1>
    
    <?php
    echo "<p>Hello from PHP running in Docker!</p>";
    echo "<p>PHP Version: " . phpversion() . "</p>";
    
    // Test MySQL connection
    try {
        $host = getenv('MYSQL_HOST');
        $user = getenv('MYSQL_USER');
        $pass = getenv('MYSQL_PASSWORD');
        $db = getenv('MYSQL_DB');
        
        $conn = new PDO("mysql:host=$host;dbname=$db", $user, $pass);
        $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
        echo "<p>Connected successfully to MySQL</p>";
    } catch(PDOException $e) {
        echo "<p>Connection failed: " . $e->getMessage() . "</p>";
    }
    ?>
</body>
</html>

    Step 8: Create a Dockerfile for Custom PHP Configuration

    If you need to install additional PHP extensions or customize the PHP environment:

    nano Dockerfile

    Add the following content:

    FROM php:8.2-apache

# Install dependencies
RUN apt-get update && apt-get install -y \
    libpng-dev \
    libjpeg-dev \
    libfreetype6-dev \
    zip \
    unzip

# Configure and install PHP extensions
RUN docker-php-ext-configure gd --with-freetype --with-jpeg \
    && docker-php-ext-install -j$(nproc) gd \
    && docker-php-ext-install pdo pdo_mysql mysqli

# Enable Apache mod_rewrite
RUN a2enmod rewrite

# Set working directory
WORKDIR /var/www/html

# Expose port 80
EXPOSE 80

    Then update your docker-compose.yml to use this Dockerfile:

      php:
    build: .
    container_name: php_dev
    # rest of the configuration remains the same

    Step 9: Start the Docker Environment

    docker compose up -d

    This starts all the services defined in docker-compose.yml in detached mode (in the background).

    Step 10: Verify Everything is Working

    1. Open http://localhost in your browser to see your PHP application
    2. Open http://localhost:8080 to access phpMyAdmin (login with root/rootpassword or phpuser/phppassword)

    Docker for Different Projects

    One of the biggest advantages of Docker is that you can have different environments for different projects. Each project can have its own docker-compose.yml with different PHP versions, extensions, and configurations.

    Analogy: Using Docker is like having separate, self-contained kitchens for each type of cuisine you want to cook. No more worrying about ingredient conflicts—your Italian kitchen won't interfere with your Thai kitchen, and you can easily switch between them or have both running simultaneously.

    Development Environment Comparison for Ubuntu

    The comparison above visualizes the trade-offs between different development environment approaches. XAMPP offers simplicity but less production parity, individual components (LAMP) offer better native integration but require more configuration, and Docker provides the best production parity but with a steeper learning curve.

    Recommendations

    • For beginners: Start with XAMPP to get familiar with PHP development quickly.
    • For learning server administration: Try the individual components (LAMP) approach to understand how each piece works.
    • For professional development: Invest time in learning Docker, as it provides the best workflow for modern PHP development.
    • For teams: Docker is the best choice for ensuring consistent environments across team members.

    Common Issues and Troubleshooting

    XAMPP Issues on Ubuntu

    • Port conflicts: Apache may not start because port 80 is already in use 
      # Find what's using port 80
sudo lsof -i :80

# Change XAMPP's Apache port
sudo nano /opt/lampp/etc/httpd.conf

# Look for "Listen 80" and change it to another port, e.g., "Listen 8080"
# Then restart XAMPP
    • Permission issues: Can't write to the htdocs directory 
      # Change ownership of the htdocs directory
sudo chown -R $USER:$USER /opt/lampp/htdocs
    • MySQL not starting: Usually due to port conflicts or corrupt data 
      # Check if another MySQL instance is running
sudo lsof -i :3306

# If MySQL data is corrupt, you might need to reset it (warning: this deletes all data)
sudo rm -rf /opt/lampp/var/mysql/
sudo /opt/lampp/lampp restart

    LAMP Stack Issues

    • Apache not starting: Check for configuration errors 
      # Test Apache configuration
sudo apache2ctl configtest

# Check Apache error log
sudo tail -f /var/log/apache2/error.log
    • PHP modules not working: Make sure they're installed and enabled 
      # List installed PHP modules
php -m

# Install a missing module (example: GD library)
sudo apt install php-gd

# Restart Apache after installing new modules
sudo systemctl restart apache2
    • MySQL connection issues: Check credentials and permissions 
      # Try connecting to MySQL from command line
mysql -u root -p

# Check MySQL error log
sudo tail -f /var/log/mysql/error.log

    Docker Issues

    • Container not starting: Check logs for errors 
      # View logs for a specific container
docker logs php_dev
    • Volume mounting issues: Check paths and permissions 
      # Verify volume mounts
docker inspect php_dev | grep -A 10 Mounts
    • Network issues: Check container networking 
      # List networks
docker network ls

# Inspect network
docker network inspect bridge
    • Permission denied errors: Might need to fix file ownership 
      # Access container shell
docker exec -it php_dev bash

# Inside container, check owner of files
ls -la /var/www/html

# If needed, change ownership
chown -R www-data:www-data /var/www/html

    Best Practices for PHP Development on Ubuntu

    Security

    • Change default passwords: Always change default passwords for MySQL, phpMyAdmin, etc.
    • Use environment variables: Store sensitive information in environment variables, not in code
    • Regular updates: Keep your development environment updated to patch security vulnerabilities
    • Development-only access: Limit development environment access to localhost or your internal network

    Performance

    • Optimize PHP configuration: Adjust php.ini settings for development performance
    • Enable opcache: Speed up PHP execution with opcache 
      # Install opcache
sudo apt install php-opcache

# Enable and configure in php.ini
opcache.enable=1
opcache.memory_consumption=128
opcache.interned_strings_buffer=8
opcache.max_accelerated_files=4000
    • Use the right PHP version: Match your development and production PHP versions

    Workflow

    • Version control: Use Git for all projects
    • Project isolation: Keep different projects separate, especially if they require different PHP versions
    • Use Composer: Manage PHP dependencies with Composer 
      # Install Composer globally
sudo apt install composer

# Initialize a project
composer init

# Install dependencies
composer install
    • Error reporting: Enable detailed error reporting during development 
      # Edit php.ini
error_reporting = E_ALL
display_errors = On
display_startup_errors = On
log_errors = On

    Development Tools

    • IDE/Editor: Use a good PHP IDE or code editor (PHPStorm, VSCode with PHP extensions, etc.)
    • Debugging: Set up Xdebug for step-through debugging 
      # Install Xdebug
sudo apt install php-xdebug

# Configure Xdebug in php.ini
zend_extension=xdebug.so
xdebug.mode=debug
xdebug.start_with_request=yes
xdebug.client_host=127.0.0.1
xdebug.client_port=9003
    • Code quality: Use PHP_CodeSniffer, PHPStan, or other code quality tools

    Project: Task Manager

    We'll create a basic task manager that allows adding and viewing tasks.

    Database Structure

    CREATE TABLE tasks (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

    Project Files

    We'll need three main files:

    1. index.php - List all tasks
    2. add_task.php - Form to add a new task
    3. db_connect.php - Database connection script

    Implementation Steps

    For each environment (XAMPP, LAMP, Docker), follow these steps:

    1. Set up the project directory
      • XAMPP: Create a folder in /opt/lampp/htdocs/task_manager
      • LAMP: Create a folder in /var/www/html/task_manager
      • Docker: Create a folder in your project's src directory
    2. Create the database connection file (db_connect.php) 
      <?php
// Database connection parameters
$host = 'localhost'; // For Docker, use the service name: 'mysql'
$dbname = 'task_manager';
$username = 'root'; // Use appropriate credentials
$password = ''; // Use appropriate password

// Create connection
try {
    $pdo = new PDO("mysql:host=$host;dbname=$dbname", $username, $password);
    // Set the PDO error mode to exception
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
} catch(PDOException $e) {
    die("ERROR: Could not connect. " . $e->getMessage());
}
?>
    3. Create the task list page (index.php) 
      <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Task Manager</title>
    <style>
        body { font-family: Arial, sans-serif; margin: 0; padding: 20px; }
        h1 { color: #333; }
        .task { border: 1px solid #ddd; padding: 10px; margin-bottom: 10px; border-radius: 5px; }
        .task h3 { margin-top: 0; }
        .add-button { display: inline-block; background: #4CAF50; color: white; padding: 10px 15px; 
                     text-decoration: none; border-radius: 5px; margin-bottom: 20px; }
    </style>
</head>
<body>
    <h1>Task Manager</h1>
    <a href="add_task.php" class="add-button">Add New Task</a>
    
    <div class="tasks">
        <?php
        // Include database connection
        require_once "db_connect.php";
        
        // Fetch tasks from database
        try {
            $sql = "SELECT * FROM tasks ORDER BY created_at DESC";
            $stmt = $pdo->query($sql);
            
            if($stmt->rowCount() > 0) {
                while($row = $stmt->fetch()) {
                    echo '<div class="task">';
                    echo '<h3>' . htmlspecialchars($row['title']) . '</h3>';
                    echo '<p>' . htmlspecialchars($row['description']) . '</p>';
                    echo '<small>Created: ' . $row['created_at'] . '</small>';
                    echo '</div>';
                }
            } else {
                echo '<p>No tasks found.</p>';
            }
        } catch(PDOException $e) {
            die("ERROR: Could not execute query. " . $e->getMessage());
        }
        ?>
    </div>
</body>
</html>
    4. Create the task form page (add_task.php) 
      <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Add Task</title>
    <style>
        body { font-family: Arial, sans-serif; margin: 0; padding: 20px; }
        h1 { color: #333; }
        form { max-width: 500px; }
        label { display: block; margin-top: 10px; }
        input[type="text"], textarea { width: 100%; padding: 8px; margin-top: 5px; }
        textarea { height: 100px; }
        button { background: #4CAF50; color: white; padding: 10px 15px; 
                border: none; border-radius: 5px; margin-top: 10px; cursor: pointer; }
        .back-link { display: inline-block; margin-top: 20px; }
    </style>
</head>
<body>
    <h1>Add New Task</h1>
    
    <?php
    // Process form submission
    if($_SERVER["REQUEST_METHOD"] == "POST") {
        // Include database connection
        require_once "db_connect.php";
        
        // Get form data
        $title = trim($_POST["title"]);
        $description = trim($_POST["description"]);
        
        // Validate input
        if(empty($title)) {
            $error = "Please enter a task title.";
        } else {
            // Insert task into database
            try {
                $sql = "INSERT INTO tasks (title, description) VALUES (:title, :description)";
                $stmt = $pdo->prepare($sql);
                
                // Bind parameters
                $stmt->bindParam(":title", $title);
                $stmt->bindParam(":description", $description);
                
                // Execute the query
                if($stmt->execute()) {
                    // Redirect to task list
                    header("location: index.php");
                    exit();
                } else {
                    $error = "Something went wrong. Please try again later.";
                }
            } catch(PDOException $e) {
                $error = "ERROR: Could not execute query. " . $e->getMessage();
            }
        }
    }
    ?>
    
    <?php if(isset($error)): ?>
        <p style="color: red;"><?php echo $error; ?></p>
    <?php endif; ?>
    
    <form action="<?php echo htmlspecialchars($_SERVER["PHP_SELF"]); ?>" method="post">
        <div>
            <label for="title">Task Title</label>
            <input type="text" name="title" id="title" required>
        </div>
        
        <div>
            <label for="description">Description</label>
            <textarea name="description" id="description"></textarea>
        </div>
        
        <button type="submit">Add Task</button>
    </form>
    
    <a href="index.php" class="back-link">Back to Task List</a>
</body>
</html>

      • LAMP: Visit http://localhost/task_manager

    • Individual Components (LAMP): A more native Ubuntu approach with greater control
    • Docker: A modern containerized approach for the best isolation and consistency

    Each approach has its strengths and is suitable for different scenarios. As you grow as a developer, you might find yourself moving from the simplicity of XAMPP to the power and flexibility of Docker, especially for team-based and production-like environments.

    Remember that the goal of any development environment is to enable you to write, test, and debug code efficiently. Choose the approach that best fits your current needs and skill level, but don't be afraid to explore other options as your requirements evolve.

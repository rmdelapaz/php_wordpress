HTML Document Structure and Syntax

Duration: 45 minutes
Module 1, Session 2

Learning Objectives

  • Understand the anatomy of HTML elements
  • Master HTML attributes and their usage
  • Learn proper nesting and indentation
  • Understand block vs inline elements
  • Master semantic HTML5 structure
  • Learn HTML entity codes

Introduction

HTML is the foundation of every webpage. Understanding its structure and syntax is crucial for building well-organized, accessible, and maintainable websites. Today, we'll dive deep into how HTML really works.

Anatomy of an HTML Element

Every HTML element follows a specific structure. Let's break it down:

<tagname attribute="value">Content goes here</tagname>
│         │                    │                 │
│         │                    │                 └── Closing tag
│         │                    └──────────────────── Content
│         └───────────────────────────────────────── Attribute
└─────────────────────────────────────────────────── Opening tag

Types of HTML Elements

Container Elements

Have opening and closing tags

<div>Content</div>
<p>Paragraph</p>
<h1>Heading</h1>

Self-Closing Elements

No closing tag needed

<img src="image.jpg" alt="Description">
<br>
<hr>
<input type="text">

HTML Attributes

Attributes provide additional information about elements. They always go in the opening tag.

Common Global Attributes

Attribute Purpose Example
id Unique identifier <div id="header">
class CSS styling hook <p class="intro text-large">
style Inline CSS <p style="color: blue;">
title Tooltip text <abbr title="World Health Organization">WHO</abbr>
lang Language specification <p lang="es">Hola</p>
data-* Custom data storage <div data-user-id="123">
💡
Attribute Rules
  • Always use quotes around attribute values
  • Use lowercase for attribute names
  • Boolean attributes can be written without values: <input disabled>
  • Order doesn't matter, but be consistent

Proper Nesting and Indentation

Proper nesting is crucial for valid HTML. Elements must be closed in the reverse order they were opened.

❌ Incorrect Nesting

<p>This is <strong>bold text</p></strong>

<ul>
<li>Item 1
<li>Item 2
</ul>

✅ Correct Nesting

<p>This is <strong>bold text</strong></p>

<ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>

Indentation Best Practices

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Page Title</title>
  </head>
  <body>
    <header>
      <nav>
        <ul>
          <li><a href="#">Home</a></li>
          <li><a href="#">About</a></li>
        </ul>
      </nav>
    </header>
    <main>
      <article>
        <h1>Article Title</h1>
        <p>Article content...</p>
      </article>
    </main>
  </body>
</html>

Block vs Inline Elements

Understanding the difference between block and inline elements is fundamental to HTML layout.

Block Elements

  • Start on a new line
  • Take full width available
  • Can contain other elements
  • Respect width/height properties

Examples:

<div> <p> <h1>-<h6> 
<ul> <ol> <li>
<header> <footer> <main>
<section> <article> <aside>
<form> <table>

Inline Elements

  • Don't start on new line
  • Only take necessary width
  • Cannot contain block elements
  • Don't respect width/height

Examples:

<span> <a> <strong> <em>
<img> <input> <button>
<label> <code> <br>
<small> <sub> <sup>

Semantic HTML5 Structure

Semantic HTML uses elements that describe their meaning, improving accessibility and SEO.

Document Structure Elements

<body>
  <header>
    <!-- Site header: logo, navigation, etc. -->
    <nav>
      <!-- Navigation menu -->
    </nav>
  </header>
  
  <main>
    <!-- Main content (only one per page) -->
    <article>
      <!-- Self-contained content -->
      <header>
        <h1>Article Title</h1>
        <time datetime="2025-01-15">January 15, 2025</time>
      </header>
      
      <section>
        <!-- Thematic grouping of content -->
        <h2>Section Title</h2>
        <p>Content...</p>
      </section>
      
      <aside>
        <!-- Related but separate content -->
        <h3>Related Links</h3>
      </aside>
    </article>
  </main>
  
  <footer>
    <!-- Site footer: copyright, links, etc. -->
  </footer>
</body>

Why Use Semantic HTML?

  • Accessibility: Screen readers understand content structure
  • SEO: Search engines better understand your content
  • Maintainability: Code is self-documenting
  • Future-proof: New devices can interpret content correctly

HTML Entity Codes

Special characters in HTML need to be encoded to display correctly:

Character Entity Name Entity Number Description
< &lt; &#60; Less than
> &gt; &#62; Greater than
& &amp; &#38; Ampersand
" &quot; &#34; Double quote
' &apos; &#39; Single quote
  &nbsp; &#160; Non-breaking space
© &copy; &#169; Copyright
® &reg; &#174; Registered
™ &trade; &#8482; Trademark

Comments in HTML

Comments are notes in your code that aren't displayed on the page:

<!-- This is a single-line comment -->

<!-- 
  This is a multi-line comment.
  It can span multiple lines.
  Great for documentation!
-->

<!-- TODO: Add more content here -->
<!-- NOTE: This section needs review -->
<!-- FIXME: Broken link below -->
⚠️
Comment Caution
Comments are visible in the page source! Never put sensitive information (passwords, API keys, etc.) in HTML comments.

Complete Example

Here's a complete HTML document showcasing proper structure and syntax:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="Learn proper HTML structure and syntax">
    <meta name="author" content="Your Name">
    <title>HTML Structure Example</title>
</head>
<body>
    <!-- Site Header -->
    <header>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
                <li><a href="#services">Services</a></li>
                <li><a href="#contact">Contact</a></li>
            </ul>
        </nav>
    </header>

    <!-- Main Content -->
    <main>
        <article>
            <header>
                <h1>Understanding HTML Structure</h1>
                <p>Published on <time datetime="2025-01-15">January 15, 2025</time></p>
            </header>
            
            <section>
                <h2>Introduction</h2>
                <p>HTML provides the <strong>structural foundation</strong> of web pages.</p>
                <p>It uses <em>semantic elements</em> to describe content meaning.</p>
            </section>
            
            <section>
                <h2>Key Concepts</h2>
                <ul>
                    <li>Elements and tags</li>
                    <li>Attributes and values</li>
                    <li>Document structure</li>
                </ul>
            </section>
            
            <aside>
                <h3>Quick Tip</h3>
                <p>Always validate your HTML using the W3C validator!</p>
            </aside>
        </article>
    </main>

    <!-- Site Footer -->
    <footer>
        <p>&copy; 2025 Your Website. All rights reserved.</p>
        <address>
            Contact us at <a href="mailto:info@example.com">info@example.com</a>
        </address>
    </footer>
</body>
</html>

WordPress Context

How This Relates to WordPress

Understanding HTML structure is crucial for WordPress development:

  • Theme Development: WordPress themes generate HTML dynamically using PHP
  • Content Editor: The WordPress block editor creates semantic HTML
  • Customization: You'll often need to modify HTML output in themes and plugins
  • SEO: Proper HTML structure improves WordPress site SEO

Example of WordPress-generated HTML:

<article id="post-123" class="post-123 post type-post status-publish">
    <header class="entry-header">
        <h1 class="entry-title">Post Title</h1>
        <div class="entry-meta">
            <time class="entry-date" datetime="2025-01-15">January 15, 2025</time>
        </div>
    </header>
    <div class="entry-content">
        <!-- Post content -->
    </div>
</article>

Practice Exercise

Create a semantic HTML document that includes:

  1. Proper DOCTYPE and language declaration
  2. Complete head section with meta tags
  3. Semantic structural elements (header, nav, main, article, section, aside, footer)
  4. At least 3 different types of global attributes
  5. Both block and inline elements
  6. Properly nested elements with consistent indentation
  7. HTML comments explaining each section
  8. At least 5 different HTML entities
📝
Challenge
Create a "Blog Post" template using semantic HTML that could be used for a WordPress theme!

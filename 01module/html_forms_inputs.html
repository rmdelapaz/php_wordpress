HTML Forms and Inputs

Duration: 45 minutes
Module 1, Session 2

Learning Objectives

  • Understand HTML form structure and elements
  • Master various input types and their uses
  • Learn form attributes and validation
  • Implement accessibility best practices for forms
  • Build user-friendly forms for data collection
  • Understand form submission methods (GET vs POST)

Introduction to HTML Forms

Forms are essential for collecting user input on the web. From simple contact forms to complex registration systems, forms enable interaction between users and websites.

Why Forms Matter
Forms are the gateway to user interaction. They power login systems, e-commerce checkouts, surveys, and much more.

Basic Form Structure

Every form starts with the <form> element:

<form action="/submit" method="POST">
    <!-- Form elements go here -->
</form>

Form Attributes

  • action - Where to send form data
  • method - HTTP method (GET or POST)
  • enctype - Encoding type for file uploads
  • name - Form name for JavaScript reference
  • target - Where to display response

Text Input Types

<!-- Basic text input -->
<label for="username">Username:</label>
<input type="text" id="username" name="username" required>

<!-- Email input -->
<label for="email">Email:</label>
<input type="email" id="email" name="email" required>

<!-- Password input -->
<label for="password">Password:</label>
<input type="password" id="password" name="password" required>

<!-- Number input -->
<label for="age">Age:</label>
<input type="number" id="age" name="age" min="1" max="120">

<!-- Tel input -->
<label for="phone">Phone:</label>
<input type="tel" id="phone" name="phone" pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}">

<!-- URL input -->
<label for="website">Website:</label>
<input type="url" id="website" name="website">

<!-- Search input -->
<label for="search">Search:</label>
<input type="search" id="search" name="search">

HTML5 Form Attributes

Validation Attributes

  • required - Field must be filled
  • pattern - Regular expression pattern
  • min/max - Minimum/maximum values
  • minlength/maxlength - Character limits
  • step - Legal number intervals

Other Useful Attributes

  • placeholder - Hint text
  • autofocus - Auto focus on load
  • autocomplete - Browser autocomplete
  • readonly - Read-only field
  • disabled - Disabled field
<input type="email" 
       name="email" 
       placeholder="your@email.com"
       required
       autofocus
       autocomplete="email">

Complete Form Example

<form action="/register" method="POST">
    <h2>Registration Form</h2>
    
    <!-- Personal Information -->
    <fieldset>
        <legend>Personal Information</legend>
        
        <div class="form-group">
            <label for="fullname">Full Name:</label>
            <input type="text" id="fullname" name="fullname" required>
        </div>
        
        <div class="form-group">
            <label for="email">Email:</label>
            <input type="email" id="email" name="email" required>
        </div>
        
        <div class="form-group">
            <label for="password">Password:</label>
            <input type="password" id="password" name="password" 
                   minlength="8" required>
        </div>
        
        <div class="form-group">
            <label for="birthdate">Date of Birth:</label>
            <input type="date" id="birthdate" name="birthdate">
        </div>
    </fieldset>
    
    <!-- Preferences -->
    <fieldset>
        <legend>Preferences</legend>
        
        <div class="form-group">
            <label for="newsletter">
                <input type="checkbox" id="newsletter" name="newsletter">
                Subscribe to newsletter
            </label>
        </div>
        
        <div class="form-group">
            <label for="terms">
                <input type="checkbox" id="terms" name="terms" required>
                I agree to the terms and conditions
            </label>
        </div>
    </fieldset>
    
    <!-- Submit Button -->
    <button type="submit">Register</button>
    <button type="reset">Clear Form</button>
</form>

Form Accessibility

Accessibility Best Practices

  • Always use <label> elements
  • Use <fieldset> and <legend> for groups
  • Provide clear error messages
  • Ensure keyboard navigation works
  • Use ARIA attributes when needed
  • Test with screen readers
<!-- Accessible form with ARIA -->
<div role="group" aria-labelledby="shipping-label">
    <h3 id="shipping-label">Shipping Address</h3>
    
    <label for="street">Street Address:
        <span aria-label="required">*</span>
    </label>
    <input type="text" id="street" name="street" 
           required aria-required="true"
           aria-describedby="street-error">
    <span id="street-error" class="error" aria-live="polite"></span>
</div>

WordPress Context

Forms in WordPress

In WordPress development, forms are crucial for:

  • Contact Forms: Contact Form 7, WPForms
  • Comments: Built-in comment forms
  • User Registration: Custom registration forms
  • Search: Search forms in themes
  • Custom Post Types: Admin forms for content

Example WordPress search form:

<form role="search" method="get" action="<?php echo home_url('/'); ?>">
    <label>
        <span class="screen-reader-text">Search for:</span>
        <input type="search" class="search-field" 
               placeholder="Search..." 
               value="<?php echo get_search_query(); ?>" 
               name="s">
    </label>
    <button type="submit">Search</button>
</form>

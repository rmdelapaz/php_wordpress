HTML Forms and Inputs

Duration: 45 minutes
Module 1: HTML Fundamentals

Learning Objectives

  • Understand HTML structure and semantics
  • Create well-formed HTML documents
  • Use HTML tags effectively
  • Build accessible web content

The Power of User Interaction

HTML forms are the primary way websites collect information from users. From simple contact forms to complex e-commerce checkout processes, forms enable two-way communication between websites and their visitors. Today, we'll explore forms and input elements to create interactive web experiences.

The Conversation Analogy

Think of HTML forms as a conversation between your website and its visitors:

  • The form itself is like starting a conversation with a purpose
  • Labels are like asking specific questions
  • Input fields are like spaces where the user can respond
  • Submit buttons are like saying "I'm done talking, now it's your turn to respond"
  • Server-side processing is like considering what the user said and deciding how to respond

Just as a good conversation flows naturally and makes sense to both parties, a well-designed form should be intuitive and easy to use.

The Anatomy of an HTML Form

Basic Form Structure

<form action="process.php" method="post">
    <label for="username">Username:</label>
    <input type="text" id="username" name="username" required>
    
    <label for="password">Password:</label>
    <input type="password" id="password" name="password" required>
    
    <button type="submit">Log In</button>
</form>

Essential Form Elements

The Form Element

The <form> element is the container for all form elements. It has several important attributes:

  • action - The URL where form data is sent (a server-side script)
  • method - How the data is sent:
    • get - Data is appended to the URL (visible, limited in size, bookmarkable)
    • post - Data is sent in the HTTP request body (not visible in URL, no size limitation, more secure)
  • autocomplete - Controls browser autocomplete behavior (on or off)
  • novalidate - Disables browser validation (use when implementing custom validation)
  • enctype - How the data is encoded:
    • application/x-www-form-urlencoded - Default encoding
    • multipart/form-data - Required for file uploads
    • text/plain - Plain text, rarely used

Labels

The <label> element provides a text description for an input field. This is crucial for usability and accessibility.

<!-- Method 1: Using the "for" attribute to connect with input ID -->
<label for="email">Email Address:</label>
<input type="email" id="email" name="email">

<!-- Method 2: Wrapping the input with the label -->
<label>
    Email Address:
    <input type="email" name="email">
</label>
Best Practices for Labels:
  • Always include labels for form controls
  • Use the for attribute that matches the input's id
  • Make labels clear and concise
  • Position labels consistently (above or to the left of inputs)

Fieldset and Legend

Use <fieldset> to group related form controls, and <legend> to provide a caption for the group.

<fieldset>
    <legend>Contact Information</legend>
    
    <label for="name">Name:</label>
    <input type="text" id="name" name="name">
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email">
    
    <label for="phone">Phone:</label>
    <input type="tel" id="phone" name="phone">
</fieldset>

Fieldsets are especially useful for longer forms with logical sections, and they improve accessibility by grouping related controls together.

Text Input Types

<!-- Basic text input -->
<label for="name">Name:</label>
<input type="text" id="name" name="name" placeholder="Enter your full name">

<!-- Password input (masked characters) -->
<label for="password">Password:</label>
<input type="password" id="password" name="password" placeholder="Enter your password">

<!-- Email input with validation -->
<label for="email">Email:</label>
<input type="email" id="email" name="email" placeholder="name@example.com">

<!-- Phone number input -->
<label for="phone">Phone:</label>
<input type="tel" id="phone" name="phone" placeholder="(123) 456-7890">

<!-- Search input (often has a clear button) -->
<label for="search">Search:</label>
<input type="search" id="search" name="search" placeholder="Search...">

<!-- URL input with validation -->
<label for="website">Website:</label>
<input type="url" id="website" name="website" placeholder="https://example.com">

<!-- Number input with constraints -->
<label for="age">Age:</label>
<input type="number" id="age" name="age" min="0" max="120" step="1">

Common Attributes for Text Inputs:

  • placeholder - Hint text displayed when the field is empty
  • value - Default value
  • size - Visual width of the input
  • maxlength - Maximum number of characters allowed
  • minlength - Minimum number of characters required
  • pattern - Regular expression for validation
  • required - Makes the field mandatory
  • readonly - Makes the field read-only (cannot be edited)
  • disabled - Disables the field completely
  • autocomplete - Controls browser autocomplete behavior

Real-World Applications:

  • text - Names, addresses, general text input
  • password - Login/registration forms
  • email - Contact forms, account creation
  • tel - Contact forms, checkout forms
  • search - Site search functionality
  • url - Social media profiles, portfolio links
  • number - Age, quantity, statistical information

Selection Input Types

Checkboxes

Checkboxes allow users to select multiple options from a set.

<h4>Which technologies do you use?</h4>
<div>
    <input type="checkbox" id="html" name="tech" value="html">
    <label for="html">HTML</label>
</div>
<div>
    <input type="checkbox" id="css" name="tech" value="css">
    <label for="css">CSS</label>
</div>
<div>
    <input type="checkbox" id="js" name="tech" value="js">
    <label for="js">JavaScript</label>
</div>
<div>
    <input type="checkbox" id="php" name="tech" value="php">
    <label for="php">PHP</label>
</div>

Note that each checkbox has the same name attribute but different value attributes. When the form is submitted, only the checked options will be sent, with their respective values.

Radio Buttons

Radio buttons allow users to select exactly one option from a set.

<h4>What is your preferred contact method?</h4>
<div>
    <input type="radio" id="contact-email" name="contact" value="email" checked>
    <label for="contact-email">Email</label>
</div>
<div>
    <input type="radio" id="contact-phone" name="contact" value="phone">
    <label for="contact-phone">Phone</label>
</div>
<div>
    <input type="radio" id="contact-text" name="contact" value="text">
    <label for="contact-text">Text Message</label>
</div>

All radio buttons in a group must have the same name attribute but different value attributes. When the form is submitted, only the value of the selected option will be sent.

Select Menus (Dropdowns)

Select menus allow users to choose one or multiple options from a dropdown list.

<!-- Single selection dropdown -->
<label for="country">Country:</label>
<select id="country" name="country">
    <option value="">Select a country</option>
    <option value="us">United States</option>
    <option value="ca">Canada</option>
    <option value="mx">Mexico</option>
    <option value="uk">United Kingdom</option>
    <option value="fr">France</option>
    <option value="jp">Japan</option>
</select>

<!-- Option groups -->
<label for="vehicle">Vehicle:</label>
<select id="vehicle" name="vehicle">
    <optgroup label="Cars">
        <option value="sedan">Sedan</option>
        <option value="suv">SUV</option>
        <option value="truck">Truck</option>
    </optgroup>
    <optgroup label="Motorcycles">
        <option value="sport">Sport</option>
        <option value="cruiser">Cruiser</option>
        <option value="touring">Touring</option>
    </optgroup>
</select>

<!-- Multiple selection dropdown -->
<label for="languages">Programming Languages:</label>
<select id="languages" name="languages[]" multiple size="5">
    <option value="html">HTML</option>
    <option value="css">CSS</option>
    <option value="js">JavaScript</option>
    <option value="php">PHP</option>
    <option value="python">Python</option>
    <option value="ruby">Ruby</option>
</select>

Notice that for multiple selections, the name attribute has square brackets ([]) to indicate it will be an array of values on the server side.

Datalist for Suggestions

The <datalist> element provides a list of predefined options for an input, creating an autocomplete experience.

<label for="browser">Choose your browser:</label>
<input list="browsers" id="browser" name="browser">
<datalist id="browsers">
    <option value="Chrome">
    <option value="Firefox">
    <option value="Safari">
    <option value="Edge">
    <option value="Opera">
</datalist>

Unlike a select menu, datalists allow users to type their own value, with suggestions appearing as they type.

Date and Time Input Types

<!-- Date picker -->
<label for="dob">Date of Birth:</label>
<input type="date" id="dob" name="dob" min="1900-01-01" max="2025-12-31">

<!-- Time picker -->
<label for="meeting-time">Meeting Time:</label>
<input type="time" id="meeting-time" name="meeting-time">

<!-- Date and time picker -->
<label for="appointment">Appointment:</label>
<input type="datetime-local" id="appointment" name="appointment">

<!-- Month picker -->
<label for="credit-expiry">Credit Card Expiry:</label>
<input type="month" id="credit-expiry" name="credit-expiry">

<!-- Week picker -->
<label for="vacation-week">Vacation Week:</label>
<input type="week" id="vacation-week" name="vacation-week">

Browser Support Note:

Date and time inputs render differently across browsers. Some browsers display native date/time pickers, while others may fall back to text inputs. Consider using JavaScript libraries for consistent date/time pickers if this is a concern.

Special Input Types

File Uploads

<!-- Basic file input -->
<label for="resume">Upload Resume:</label>
<input type="file" id="resume" name="resume">

<!-- Multiple file selection -->
<label for="photos">Upload Photos:</label>
<input type="file" id="photos" name="photos[]" multiple>

<!-- Limiting file types -->
<label for="avatar">Profile Picture:</label>
<input type="file" id="avatar" name="avatar" accept="image/png, image/jpeg">

Important: When creating a form that includes file uploads, you must include enctype="multipart/form-data" in the form tag:

<form action="upload.php" method="post" enctype="multipart/form-data">
    <!-- File inputs here -->
</form>

Color Picker

<label for="theme-color">Choose Theme Color:</label>
<input type="color" id="theme-color" name="theme-color" value="#3498db">

Range Slider

<label for="volume">Volume:</label>
<input type="range" id="volume" name="volume" min="0" max="100" step="1" value="50">

Range sliders are often paired with output elements or JavaScript to display the current value:

<label for="price-range">Price Range: <span id="price-value">$50</span></label>
<input type="range" id="price-range" name="price" min="0" max="100" step="5" value="50" 
       oninput="document.getElementById('price-value').textContent = '$' + this.value">

Hidden Inputs

Hidden inputs store data that isn't displayed to the user but is submitted with the form.

<input type="hidden" id="user-id" name="user-id" value="123456">
<input type="hidden" id="form-token" name="form-token" value="abc123xyz">

Common uses include storing session IDs, CSRF tokens, user IDs, or other data needed for processing but not visible to the user.

Buttons and Form Submission

There are multiple ways to create buttons in forms:

<!-- Submit button using input -->
<input type="submit" value="Submit Form">

<!-- Reset button (clears form fields) -->
<input type="reset" value="Clear Form">

<!-- Generic button (functionality added with JavaScript) -->
<input type="button" value="Calculate" onclick="calculateTotal()">

<!-- Button element (preferred modern approach) -->
<button type="submit">Submit Form</button>
<button type="reset">Clear Form</button>
<button type="button" onclick="calculateTotal()">Calculate</button>

Advantages of <button> over <input type="button">:

  • Can contain HTML content (images, text formatting, etc.)
  • Easier to style
  • More semantic (better represents its purpose)

Textareas for Multi-line Text

The <textarea> element creates a multi-line text input field, perfect for comments, messages, and longer text content.

<label for="comments">Additional Comments:</label>
<textarea id="comments" name="comments" rows="5" cols="40" placeholder="Please share any additional information..."></textarea>

Important Textarea Attributes:

  • rows - Number of visible text lines
  • cols - Width in average character widths
  • wrap - Text wrapping behavior (soft, hard, or off)
  • maxlength - Maximum number of characters
  • minlength - Minimum number of characters
  • placeholder - Hint text when empty
  • required - Makes the field mandatory
  • readonly - Makes the field read-only
  • disabled - Disables the field

Important Note About Textareas:

Unlike most HTML elements, any content between the <textarea> and </textarea> tags becomes the default value of the textarea:

<textarea id="terms" name="terms" rows="4" cols="50">
I agree to the terms and conditions outlined in the user agreement.
</textarea>

This is different from using the value attribute, which doesn't work with textareas.

Form Validation

Validation ensures that users submit data in the correct format, improving data quality and user experience.

HTML5 Built-in Validation

HTML5 introduced several validation features that work without JavaScript:

<form>
    <!-- Required field -->
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required>
    
    <!-- Email validation -->
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <!-- Minimum and maximum length -->
    <label for="username">Username (5-15 characters):</label>
    <input type="text" id="username" name="username" minlength="5" maxlength="15" required>
    
    <!-- Number range -->
    <label for="age">Age (18-120):</label>
    <input type="number" id="age" name="age" min="18" max="120" required>
    
    <!-- Pattern matching (regular expression) -->
    <label for="zip">Zip Code (5 digits):</label>
    <input type="text" id="zip" name="zip" pattern="[0-9]{5}" title="Five digit zip code" required>
    
    <button type="submit">Submit</button>
</form>

Key Validation Attributes:

  • required - Field must not be empty
  • minlength / maxlength - Text length constraints
  • min / max - Numeric value constraints
  • pattern - Regular expression pattern
  • title - Custom validation message (displayed as a tooltip)

JavaScript Validation

For more complex validation or custom error messages, JavaScript can be used:

<form id="registration-form" novalidate>
    <label for="password">Password:</label>
    <input type="password" id="password" name="password">
    <span id="password-error" class="error"></span>
    
    <label for="confirm-password">Confirm Password:</label>
    <input type="password" id="confirm-password" name="confirm-password">
    <span id="confirm-error" class="error"></span>
    
    <button type="submit">Register</button>
</form>

<script>
    document.getElementById('registration-form').addEventListener('submit', function(event) {
        // Get form fields
        const password = document.getElementById('password');
        const confirmPassword = document.getElementById('confirm-password');
        const passwordError = document.getElementById('password-error');
        const confirmError = document.getElementById('confirm-error');
        
        // Reset error messages
        passwordError.textContent = '';
        confirmError.textContent = '';
        
        // Validate password strength
        if (password.value.length < 8) {
            passwordError.textContent = 'Password must be at least 8 characters long';
            event.preventDefault(); // Prevent form submission
        }
        
        // Check if passwords match
        if (password.value !== confirmPassword.value) {
            confirmError.textContent = 'Passwords do not match';
            event.preventDefault(); // Prevent form submission
        }
    });
</script>

Note about novalidate:

The novalidate attribute on the form disables browser validation, allowing us to implement custom validation with JavaScript. This is useful when you want complete control over validation behavior and error messages.

Validation Best Practices:

  • Always validate on both client-side (browser) and server-side
  • Provide clear, specific error messages
  • Position error messages close to the problematic field
  • Use visual cues (colors, icons) to indicate valid/invalid fields
  • Validate in real-time when possible (e.g., as the user types or when a field loses focus)
  • Don't be too restrictive - accept various formats when possible (e.g., phone numbers with or without dashes)

Form Design Tips:

  • Use a single-column layout for most forms
  • Group related fields together
  • Place labels above inputs (for mobile-friendliness)
  • Use consistent spacing between form elements
  • Make buttons large enough to be easily clickable
  • Use color to indicate state (focus, error, success)
  • Ensure sufficient contrast for readability
  • Design for mobile first (responsive forms)

Form Accessibility

Creating accessible forms ensures that all users, including those with disabilities, can successfully use your forms.

Key Accessibility Practices:

  • Use proper labels - Every input should have a label that's programmatically associated with it
  • Add descriptive instructions - Provide clear guidance on how to complete the form
  • Use fieldsets and legends - Group related controls and provide a heading
  • Indicate required fields - Clearly mark which fields are mandatory
  • Provide feedback - Communicate errors in a way that's perceivable to all users
  • Maintain keyboard navigation - Ensure the form can be completed using only a keyboard
  • Add ARIA attributes when HTML semantics aren't sufficient
<!-- Accessible form example -->
<form action="process.php" method="post">
    <div class="form-instructions">
        <p>Fields marked with an asterisk (*) are required.</p>
    </div>
    
    <div class="form-group">
        <label for="full-name">Full Name *</label>
        <input type="text" id="full-name" name="full-name" required 
               aria-required="true" aria-describedby="name-help">
        <span id="name-help" class="help-text">Enter your first and last name.</span>
    </div>
    
    <fieldset>
        <legend>Contact Preferences</legend>
        
        <div class="checkbox-group">
            <input type="checkbox" id="contact-email" name="contact" value="email">
            <label for="contact-email">Email</label>
        </div>
        
        <div class="checkbox-group">
            <input type="checkbox" id="contact-phone" name="contact" value="phone">
            <label for="contact-phone">Phone</label>
        </div>
    </fieldset>
    
    <div class="form-group">
        <label for="message">Message *</label>
        <textarea id="message" name="message" required aria-required="true"
                  rows="5" cols="40"></textarea>
    </div>
    
    <div class="form-controls">
        <button type="submit">Submit</button>
        <button type="reset">Reset</button>
    </div>
</form>

Important ARIA Attributes for Forms:

  • aria-required="true" - Indicates a required field
  • aria-describedby="id" - Associates descriptive text with a form control
  • aria-labelledby="id" - Associates a label with a form control
  • aria-invalid="true" - Indicates a field has an invalid value
  • aria-live="polite" - Used for dynamically updated error messages

        // Basic form validation
        document.getElementById('contactForm').addEventListener('submit', function(event) {
            let valid = true;
            
            // Reset error messages
            const errorElements = document.querySelectorAll('.error');
            errorElements.forEach(element => {
                element.textContent = '';
            });
            
            // Validate full name
            const fullName = document.getElementById('fullName');
            if (fullName.value.trim() === '') {
                document.getElementById('fullNameError').textContent = 'Please enter your full name';
                valid = false;
            }
            
            // Validate email
            const email = document.getElementById('email');
            const emailPattern = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
            if (!emailPattern.test(email.value)) {
                document.getElementById('emailError').textContent = 'Please enter a valid email address';
                valid = false;
            }
            
            // Validate message
            const message = document.getElementById('message');
            if (message.value.trim() === '') {
                document.getElementById('messageError').textContent = 'Please enter your message';
                valid = false;
            } else if (message.value.length < 20) {
                document.getElementById('messageError').textContent = 'Message must be at least 20 characters long';
                valid = false;
            }
            
            // Prevent form submission if validation fails
            if (!valid) {
                event.preventDefault();
                // Scroll to the first error
                const firstError = document.querySelector('.error:not(:empty)');
                if (firstError) {
                    firstError.scrollIntoView({ behavior: 'smooth', block: 'center' });
                }
            }
        });
    </script>

A Brief Introduction to Form Processing with PHP

While we'll cover PHP in detail later in the course, here's a basic example of how you might process a form on the server side:

<?php
// File: process-contact.php

// Check if form was submitted
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    // Sanitize input data
    $fullName = filter_input(INPUT_POST, 'fullName', FILTER_SANITIZE_STRING);
    $email = filter_input(INPUT_POST, 'email', FILTER_SANITIZE_EMAIL);
    $phone = filter_input(INPUT_POST, 'phone', FILTER_SANITIZE_STRING);
    $contactMethod = filter_input(INPUT_POST, 'contactMethod', FILTER_SANITIZE_STRING);
    $department = filter_input(INPUT_POST, 'department', FILTER_SANITIZE_STRING);
    $subject = filter_input(INPUT_POST, 'subject', FILTER_SANITIZE_STRING);
    $message = filter_input(INPUT_POST, 'message', FILTER_SANITIZE_STRING);
    $newsletter = isset($_POST['newsletter']) ? 'Yes' : 'No';
    
    // Validate the data
    $errors = array();
    
    if (empty($fullName)) {
        $errors[] = "Full name is required";
    }
    
    if (empty($email) || !filter_var($email, FILTER_VALIDATE_EMAIL)) {
        $errors[] = "Valid email address is required";
    }
    
    if (empty($subject)) {
        $errors[] = "Subject is required";
    }
    
    if (empty($message)) {
        $errors[] = "Message is required";
    }
    
    // If no errors, process the form
    if (empty($errors)) {
        // Here you would typically:
        // 1. Store in a database
        // 2. Send an email notification
        // 3. Log the submission
        
        // For example, sending an email
        $to = "contact@example.com";
        $emailSubject = "Contact Form: $subject";
        $emailBody = "Name: $fullName\n";
        $emailBody .= "Email: $email\n";
        $emailBody .= "Phone: $phone\n";
        $emailBody .= "Preferred Contact: $contactMethod\n";
        $emailBody .= "Department: $department\n";
        $emailBody .= "Newsletter: $newsletter\n\n";
        $emailBody .= "Message:\n$message\n";
        
        $headers = "From: $email";
        
        // Send email
        if (mail($to, $emailSubject, $emailBody, $headers)) {
            // Redirect to a thank you page
            header("Location: thank-you.php");
            exit;
        } else {
            $errors[] = "Failed to send email. Please try again later.";
        }
    }
    
    // If there were errors, you might want to:
    // 1. Display the form again with error messages
    // 2. Pre-fill the form with the submitted values
    if (!empty($errors)) {
        // Store errors in a session to display them on the form page
        session_start();
        $_SESSION['errors'] = $errors;
        $_SESSION['form_data'] = $_POST;
        
        // Redirect back to the form
        header("Location: contact.php");
        exit;
    }
}
?>

Security Considerations:

  • Always validate on the server - Never trust client-side validation alone
  • Sanitize all input - Filter inputs to prevent XSS and injection attacks
  • Use prepared statements for database queries to prevent SQL injection
  • Implement CSRF protection for forms that modify data
  • Limit file uploads - Validate file types, sizes, and scan for malware if accepting uploads
  • Consider rate limiting to prevent form spam and brute force attacks

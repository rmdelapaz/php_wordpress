Essential HTML Tags

Duration: 60 minutes
Module 1, Session 2

Learning Objectives

  • Master the use of headings (h1-h6) for content hierarchy
  • Create and format paragraphs with text styling
  • Build different types of lists (ordered, unordered, description)
  • Create hyperlinks with various attributes
  • Display images with proper accessibility
  • Build tables for structured data
  • Understand and use semantic HTML5 elements

Introduction

HTML tags are the building blocks of web content. In this lesson, we'll explore the most essential HTML tags that you'll use in every web project. These tags form the foundation of semantic, accessible, and well-structured web pages.

Remember
HTML tags describe the meaning and structure of content, not its appearance. We'll use CSS to control how things look.

Headings: Structuring Your Content

HTML provides six levels of headings, from <h1> (most important) to <h6> (least important).

<h1>Main Page Title</h1>
<h2>Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>Sub-subsection Heading</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

Best Practices for Headings

  • Use only one <h1> per page
  • Don't skip heading levels
  • Use headings for structure, not styling
  • Keep headings descriptive and concise

Paragraphs and Text Formatting

Basic Paragraphs

<p>This is a paragraph. It's the most basic way to add text content to your webpage.</p>
<p>Each paragraph is a block-level element, creating space before and after.</p>

Text Formatting Tags

Emphasis Tags

  • <strong> - Important text (bold)
  • <em> - Emphasized text (italic)
  • <mark> - Highlighted text
  • <small> - Smaller text

Editing Tags

  • <del> - Deleted text
  • <ins> - Inserted text
  • <sub> - Subscript
  • <sup> - Superscript
<p>This text contains <strong>important information</strong> and 
<em>emphasized content</em>. We can <mark>highlight</mark> key points.</p>

<p>The price was <del>$50</del> <ins>$30</ins> (40% off!)</p>

<p>The formula is H<sub>2</sub>O and E=mc<sup>2</sup></p>

Links: Connecting the Web

The <a> (anchor) tag creates hyperlinks:

<!-- External link -->
<a href="https://example.com">Visit Example.com</a>

<!-- Internal link -->
<a href="/about.html">About Us</a>

<!-- Email link -->
<a href="mailto:info@example.com">Contact Us</a>

<!-- Phone link -->
<a href="tel:+1234567890">Call Us</a>

<!-- Link opening in new tab -->
<a href="https://example.com" target="_blank" rel="noopener noreferrer">Open in New Tab</a>

<!-- Download link -->
<a href="/files/document.pdf" download>Download PDF</a>

Lists: Organizing Information

Unordered Lists

<ul>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item
        <ul>
            <li>Nested item 1</li>
            <li>Nested item 2</li>
        </ul>
    </li>
</ul>

Ordered Lists

<ol>
    <li>First step</li>
    <li>Second step</li>
    <li>Third step</li>
</ol>

<!-- Custom numbering -->
<ol start="5" type="A">
    <li>Item E</li>
    <li>Item F</li>
</ol>

Description Lists

<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>
    
    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>
    
    <dt>JavaScript</dt>
    <dd>Programming language for web interactivity</dd>
</dl>

Images: Visual Content

The <img> tag embeds images in your webpage:

<!-- Basic image -->
<img src="photo.jpg" alt="Description of the image">

<!-- Image with dimensions -->
<img src="photo.jpg" alt="Description" width="600" height="400">

<!-- Responsive image -->
<img src="photo.jpg" alt="Description" style="max-width: 100%; height: auto;">

<!-- Image with multiple sources -->
<picture>
    <source media="(min-width: 768px)" srcset="large.jpg">
    <source media="(min-width: 480px)" srcset="medium.jpg">
    <img src="small.jpg" alt="Description">
</picture>

<!-- Figure with caption -->
<figure>
    <img src="chart.jpg" alt="Sales chart for 2025">
    <figcaption>Figure 1: Quarterly sales data</figcaption>
</figure>

Image Best Practices

  • Always include descriptive alt text
  • Optimize images for web (compress file size)
  • Specify dimensions to prevent layout shift
  • Use appropriate file formats (JPEG for photos, PNG for graphics)
  • Consider lazy loading for images below the fold

Tables: Structured Data

<table>
    <caption>Product Pricing</caption>
    <thead>
        <tr>
            <th>Product</th>
            <th>Price</th>
            <th>Stock</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Widget A</td>
            <td>$19.99</td>
            <td>In Stock</td>
        </tr>
        <tr>
            <td>Widget B</td>
            <td>$29.99</td>
            <td>Limited</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td colspan="2">Total</td>
            <td>$49.98</td>
        </tr>
    </tfoot>
</table>

Table Usage
Use tables only for tabular data, not for layout. CSS Grid or Flexbox should be used for page layout.

Semantic HTML5 Elements

Semantic elements clearly describe their meaning:

<header>
    <nav>
        <ul>
            <li><a href="/">Home</a></li>
            <li><a href="/about">About</a></li>
        </ul>
    </nav>
</header>

<main>
    <article>
        <header>
            <h1>Article Title</h1>
            <time datetime="2025-01-15">January 15, 2025</time>
        </header>
        
        <section>
            <h2>Introduction</h2>
            <p>Article content...</p>
        </section>
        
        <aside>
            <h3>Related Links</h3>
            <ul>
                <li><a href="#">Related Article</a></li>
            </ul>
        </aside>
    </article>
</main>

<footer>
    <p>© 2025 Your Website</p>
</footer>

Common Semantic Elements

  • <header> - Introductory content
  • <nav> - Navigation links
  • <main> - Main content (one per page)
  • <article> - Self-contained content
  • <section> - Thematic grouping
  • <aside> - Related content
  • <footer> - Footer content

WordPress Context

How These Tags Apply to WordPress

In WordPress development, these HTML tags are generated dynamically:

  • Headings: Post titles use <h1> or <h2>
  • Lists: Navigation menus are typically unordered lists
  • Images: Featured images and media library content
  • Links: Permalinks and navigation
  • Semantic HTML: Theme structure uses semantic elements

Example WordPress-generated HTML:

<article id="post-123" class="post">
    <header class="entry-header">
        <h1 class="entry-title"><?php the_title(); ?></h1>
    </header>
    <div class="entry-content">
        <?php the_content(); ?>
    </div>
</article>

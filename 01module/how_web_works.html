How the Web Works: Servers, Browsers, and HTTP

Duration: 45 minutes
Module 1, Session 1

Learning Objectives

  • Understand the client-server model
  • Learn how browsers and servers communicate
  • Understand HTTP/HTTPS protocols
  • Learn about DNS and how URLs work
  • Understand the request-response cycle

Introduction: The Internet vs. The Web

Before diving into web development, it's crucial to understand how the web actually works. This knowledge will help you debug issues, optimize performance, and make better architectural decisions.

ℹ️
Did You Know?
The Internet and the World Wide Web are not the same thing! The Internet is the global network of connected computers, while the Web is a system of interlinked documents accessed via the Internet.

The Client-Server Model

The web operates on a client-server model. This is the foundation of how all websites work:

  • Client: Your web browser (Chrome, Firefox, Safari, etc.)
  • Server: A computer that hosts websites and responds to requests
  • Request: When you type a URL or click a link
  • Response: The HTML, CSS, JavaScript, and other files sent back
sequenceDiagram participant User participant Browser as Web Browser participant Server as Web Server User->>Browser: Types URL/Clicks Link Browser->>Server: HTTP Request Server->>Server: Process Request Server->>Browser: HTTP Response (HTML, CSS, JS) Browser->>Browser: Renders Page Browser->>User: Displays Website

A URL (Uniform Resource Locator) is the address of a resource on the web. Let's break down its components:

DNS: The Internet's Phone Book

DNS (Domain Name System) translates human-readable domain names into IP addresses that computers can understand.

graph LR A[User types example.com] --> B[Browser checks cache] B --> C[DNS Lookup] C --> D[Root Server] D --> E[TLD Server .com] E --> F[Name Server] F --> G[IP: 93.184.216.34] G --> H[Browser connects to IP]
💡
Quick Tip
You can see DNS in action by opening your terminal/command prompt and typing: nslookup google.com

HTTP: The Language of the Web

HTTP (HyperText Transfer Protocol) is how browsers and servers communicate. HTTPS is the secure version with encryption.

HTTP Request Methods

GET

Retrieve data from server

GET /products

POST

Send data to server

POST /login

PUT

Update existing resource

PUT /users/123

DELETE

Remove resource

DELETE /posts/456

HTTP Status Codes

Code Range Meaning Common Examples
1xx Informational 100 Continue
2xx Success 200 OK, 201 Created
3xx Redirection 301 Moved Permanently, 302 Found
4xx Client Error 404 Not Found, 403 Forbidden
5xx Server Error 500 Internal Server Error

The Complete Request-Response Cycle

Let's walk through what happens when you visit a website:

  1. URL Entry: You type https://example.com in your browser
  2. DNS Resolution: Browser finds the IP address (93.184.216.34)
  3. TCP Connection: Browser establishes connection with server
  4. HTTP Request: Browser sends GET request for the homepage
  5. Server Processing: Server processes request, possibly queries database
  6. HTTP Response: Server sends back HTML, with status code 200
  7. Browser Parsing: Browser reads HTML and finds linked resources
  8. Additional Requests: Browser requests CSS, JavaScript, images
  9. Rendering: Browser combines everything to display the page

Static vs Dynamic Websites

Static Websites

  • Pre-built HTML files
  • Same content for all users
  • Fast and simple
  • Limited interactivity
  • Example: Documentation sites

Dynamic Websites

  • Generated on-the-fly
  • Personalized content
  • Database-driven
  • Interactive features
  • Example: WordPress, Facebook

WordPress Context

WordPress is a dynamic website system. When someone visits a WordPress site:

  1. Server receives the request
  2. PHP processes the request
  3. WordPress queries the MySQL database
  4. PHP generates HTML from templates and data
  5. Server sends the HTML to the browser

Browser Developer Tools

Modern browsers include powerful developer tools for inspecting web traffic:

🔧
Try It Now!
  1. Press F12 or right-click and select "Inspect"
  2. Go to the "Network" tab
  3. Refresh the page
  4. Watch all the HTTP requests being made!

Practice Exercise

To reinforce your understanding of how the web works:

  1. Open your browser's developer tools (F12)
  2. Visit any website and go to the Network tab
  3. Identify:
    • The initial HTML request
    • CSS file requests
    • JavaScript file requests
    • Image requests
    • The status codes for each request
  4. Click on one request and examine the request and response headers
  5. Take a screenshot of the Network tab and save it for discussion

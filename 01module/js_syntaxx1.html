JavaScript Fundamentals: Syntax, Variables, and Data Types

A comprehensive guide for new web developers

Introduction to JavaScript

JavaScript is the language of the web—the invisible force that transforms static HTML pages into dynamic, interactive experiences. Think of HTML as the skeleton of a webpage, CSS as its appearance, and JavaScript as the muscles and nervous system that give it life and movement.

graph TD HTML[HTML - Structure] --> Webpage[Complete Webpage] CSS[CSS - Style] --> Webpage JS[JavaScript - Behavior] --> Webpage User[User Interaction] --> JS

JavaScript was created in just 10 days in 1995 by Brendan Eich while he was working at Netscape. Despite its rushed creation, it has become one of the most widely used programming languages in the world, powering everything from simple form validations to complex single-page applications.

Getting Started with JavaScript Syntax

JavaScript syntax is the set of rules that define how JavaScript programs are constructed. Like learning any language, mastering the basic grammar is essential before building more complex expressions.

JavaScript in HTML

JavaScript can be added to your HTML in three ways:

Inline JavaScript (not recommended for larger scripts):

<button onclick="alert('Hello, World!');">Click Me</button>

Internal JavaScript (in the head or body):

<script>
    // Your JavaScript code here
    console.log("Hello from internal JavaScript!");
</script>

External JavaScript (recommended for maintainability):

<script src="script.js"></script>

Analogy: JavaScript in HTML

Think of your HTML document as a book. External JavaScript is like having a separate reference guide you can use with many books. Internal JavaScript is like having footnotes specific to that book. Inline JavaScript is like writing notes in the margins—convenient but messy and hard to maintain.

JavaScript Statements and Syntax

JavaScript code is composed of statements that tell the browser what to do. Each statement typically ends with a semicolon (;), though JavaScript has automatic semicolon insertion (ASI) which can sometimes add them for you—but relying on this can lead to unexpected bugs.

// This is a single line comment

/* This is a
   multi-line comment */

// Basic statement
console.log("Hello, World!");

// Multiple statements
let name = "JavaScript"; 
console.log("Hello, " + name);

Case Sensitivity

JavaScript is case-sensitive. Variables named userName, USERNAME, and username are all different variables.

White Space and Line Breaks

JavaScript ignores spaces, tabs, and newlines that appear in JavaScript programs. You can use white space to make your code more readable.

Analogy: JavaScript Syntax

JavaScript syntax is like cooking instructions in a recipe. Each line tells you a specific action to take, and the order matters. Just as you wouldn't bake a cake before mixing the ingredients, you can't use a variable in JavaScript before declaring it.

Variables in JavaScript

Variables are containers for storing data values. In JavaScript, there are three ways to declare variables:

// Using var (older method, function scoped)
var userName = "Alice";

// Using let (modern method, block scoped)
let userAge = 25;

// Using const (for values that shouldn't change)
const PI = 3.14159;

Variable Declaration Rules

  • Variable names must begin with a letter, underscore (_), or dollar sign ($).
  • After the first character, variable names can also include numbers.
  • Variable names cannot be JavaScript reserved words like function or if.
  • Variable names are case-sensitive.

Variable Scope

Scope determines where variables are accessible in your code:

// Global scope
let globalVar = "I'm accessible everywhere";

function exampleFunction() {
    // Function scope
    let functionVar = "I'm only accessible inside this function";
    
    if (true) {
        // Block scope
        let blockVar = "I'm only accessible inside this block";
        var notBlockVar = "I'm accessible throughout the function";
    }
    
    console.log(functionVar); // Works
    console.log(notBlockVar); // Works
    console.log(blockVar); // Error: blockVar is not defined
}

console.log(globalVar); // Works
console.log(functionVar); // Error: functionVar is not defined

Analogy: Variable Scopes

Think of variable scope like rooms in a house:

  • Global variables are like announcements made over a loudspeaker—everyone in the house can hear them.
  • Function-scoped variables are like conversations in a room—only people in that room can hear them.
  • Block-scoped variables are like whispers in a corner—only people in that immediate area can hear them.

Variable Hoisting

Hoisting is JavaScript's behavior of moving declarations to the top of their scope before code execution:

// What you write
console.log(hoistedVar); // undefined (not an error!)
var hoistedVar = "I'm hoisted";

// What JavaScript actually does
var hoistedVar;
console.log(hoistedVar); // undefined
hoistedVar = "I'm hoisted";

// let and const are hoisted too, but not initialized
console.log(notHoisted); // Error: Cannot access before initialization
let notHoisted = "I'm not accessible";

Real-World Impact of Hoisting

Hoisting can lead to bugs that are difficult to track down. For example, a variable defined lower in your code might accidentally override a variable with the same name from a third-party library if you're not careful. This is one reason why modern JavaScript recommends using let and const instead of var.

Data Types in JavaScript

JavaScript is a dynamically typed language, which means you don't need to declare the type of a variable when you create it. JavaScript has seven primitive data types and one complex data type:

Primitive Data Types

String

Strings represent textual data and can be created using single quotes, double quotes, or backticks (template literals).

// String examples
let singleQuote = 'Hello, world!';
let doubleQuote = "Hello, world!";
let templateLiteral = `Hello, ${userName}!`; // Template literals allow embedding variables

In web development, strings are used everywhere—from displaying text on a webpage to storing user input, processing API responses, and manipulating DOM elements. Template literals make it particularly easy to create dynamic HTML content in JavaScript.

Number

The Number type represents both integer and floating-point numbers.

// Number examples
let integer = 42;
let float = 3.14159;
let negative = -273.15;
let exponential = 1.23e6; // 1.23 × 10^6 = 1,230,000
let binary = 0b1010; // 10
let octal = 0o744; // 484
let hexadecimal = 0xFF; // 255

// Special number values
let infinity = Infinity;
let negativeInfinity = -Infinity;
let notANumber = NaN; // Result of invalid operations

Analogy: JavaScript Numbers

Unlike some languages that have different types for integers and decimals, JavaScript's Number type is like a Swiss Army knife—it handles whole numbers, decimals, scientific notation, and even different numbering systems all in one package.

Boolean

Booleans represent logical values: true and false.

// Boolean examples
let isLoggedIn = true;
let hasPermission = false;

// Booleans often result from comparisons
let isAdult = userAge >= 18;
let isValidEmail = email.includes('@');

Undefined

A variable that has been declared but not assigned a value has the value undefined.

// Undefined example
let notDefined;
console.log(notDefined); // undefined

function sayHello(name) {
    console.log(`Hello, ${name || 'stranger'}!`);
}
sayHello(); // The parameter 'name' is undefined

Null

null represents the intentional absence of any object value. It is often used to explicitly indicate that a variable has no value.

// Null example
let emptyValue = null;
let user = null; // No user is currently logged in

// Difference between null and undefined
console.log(typeof undefined); // "undefined"
console.log(typeof null);      // "object" (this is a bug in JavaScript!)

Analogy: null vs undefined

Think of undefined as an empty shelf—the shelf exists, but there's nothing on it yet. null is like a shelf with a sign that explicitly says "Empty"—someone has deliberately marked it as containing nothing.

Symbol

Symbols were introduced in ES6 and represent unique identifiers. Every Symbol is unique, even if they have the same description.

// Symbol examples
let id1 = Symbol('id');
let id2 = Symbol('id');

console.log(id1 === id2); // false, they are different symbols

// Common use case: using symbols as object property keys
const USER_ID = Symbol('userId');
let user = {};
user[USER_ID] = 12345;

// The symbol property won't show up in regular iterations
console.log(Object.keys(user)); // []
console.log(user[USER_ID]); // 12345

Symbols are often used in libraries and frameworks for creating "private" object properties or for adding properties to objects that won't conflict with other properties, even if they have the same name.

BigInt

BigInt was added to JavaScript to represent integers larger than 2^53 - 1, which is the largest number JavaScript can represent with the Number type.

// BigInt examples
const largeNumber = 9007199254740991n; // Adding 'n' at the end makes it a BigInt
const anotherLargeNumber = BigInt("9007199254740991");

// Operations with BigInts
const result = largeNumber + 1n;
console.log(result); // 9007199254740992n

// You cannot mix BigInt and regular numbers
// console.log(largeNumber + 1); // TypeError

Reference Data Type: Object

Objects are collections of properties, and a property is an association between a name (or key) and a value.

// Object examples
let person = {
    firstName: "John",
    lastName: "Doe",
    age: 30,
    email: "john.doe@example.com",
    isEmployed: true,
    hobbies: ["reading", "hiking", "coding"],
    address: {
        street: "123 Main St",
        city: "Anytown",
        country: "Countryland"
    },
    fullName: function() {
        return this.firstName + " " + this.lastName;
    }
};

// Accessing object properties
console.log(person.firstName); // "John"
console.log(person["lastName"]); // "Doe"
console.log(person.address.city); // "Anytown"
console.log(person.fullName()); // "John Doe"

Analogy: Objects in JavaScript

If primitive data types are like single ingredients in cooking, objects are like recipes that combine multiple ingredients with instructions. They allow you to group related data and functionality together in a way that mirrors real-world entities.

Arrays

Arrays are special types of objects used for storing ordered collections of data.

// Array examples
let colors = ["red", "green", "blue"];
let mixed = [42, "hello", true, { name: "object" }, [1, 2, 3]];

// Accessing array elements (zero-indexed)
console.log(colors[0]); // "red"
console.log(mixed[3].name); // "object"

// Array methods
colors.push("yellow"); // Add to end
colors.unshift("black"); // Add to beginning
let lastColor = colors.pop(); // Remove from end
let firstColor = colors.shift(); // Remove from beginning

// Array iteration
colors.forEach(function(color) {
    console.log(color);
});

// or with arrow function
colors.map(color => color.toUpperCase());

Real-World Application: User Data Management

In web development, arrays and objects are commonly used together to manage collections of user data. For example, an e-commerce application might store a shopping cart as an array of product objects, each containing properties like name, price, quantity, and image URL.

Type Conversion and Coercion

JavaScript is loosely typed, which means variables can change types, and values can be automatically converted between types in certain operations.

Explicit Type Conversion (Type Casting)

You can manually convert between types using built-in functions:

// Converting to String
let num = 42;
let strFromNum = String(num); // "42"
let strFromBool = String(true); // "true"

// Converting to Number
let str = "42";
let numFromStr = Number(str); // 42
let numFromBool = Number(true); // 1
let notANumber = Number("hello"); // NaN

// Converting to Boolean
let boolFromNum = Boolean(42); // true (any non-zero number is true)
let boolFromStr = Boolean("hello"); // true (any non-empty string is true)
let boolFromEmpty = Boolean(""); // false
let boolFromZero = Boolean(0); // false
let boolFromNull = Boolean(null); // false
let boolFromUndefined = Boolean(undefined); // false

Implicit Type Coercion

JavaScript automatically converts types in certain operations, which can sometimes lead to unexpected results:

// String concatenation vs. numeric addition
console.log(5 + 10);     // 15 (number + number = number)
console.log(5 + "10");   // "510" (number + string = string)
console.log("5" + 10);   // "510" (string + number = string)
console.log("5" + "10"); // "510" (string + string = string)

// Other operations will try to convert strings to numbers
console.log("5" - 2);    // 3 (string is converted to number)
console.log("5" * 2);    // 10 (string is converted to number)
console.log("5" / 2);    // 2.5 (string is converted to number)

// Equality operators
console.log(5 == "5");   // true (loose equality, type coercion happens)
console.log(5 === "5");  // false (strict equality, no type coercion)
console.log(1 == true);  // true (loose equality)
console.log(1 === true); // false (strict equality)

Analogy: Type Coercion

JavaScript's type coercion is like a translator that sometimes makes creative interpretations. When you use loose equality (==), you're telling this translator: "Find a way to make these two things comparable." When you use strict equality (===), you're saying: "Only compare these if they're already speaking the same language."

Best Practice: Use Strict Equality

In professional JavaScript development, it's considered a best practice to use strict equality operators (=== and !==) instead of loose equality operators (== and !=). This avoids unexpected type coercions and makes your code more predictable and easier to debug.

The typeof Operator

The typeof operator returns a string indicating the type of an operand:

console.log(typeof "hello");    // "string"
console.log(typeof 42);        // "number"
console.log(typeof true);      // "boolean"
console.log(typeof undefined); // "undefined"
console.log(typeof Symbol());  // "symbol"
console.log(typeof 42n);       // "bigint"

// Quirks of typeof
console.log(typeof null);      // "object" (historical bug in JavaScript)
console.log(typeof []);        // "object" (arrays are objects)
console.log(typeof {});        // "object"
console.log(typeof function(){}); // "function" (functions are special objects)

Using typeof for Validation

The typeof operator is commonly used for basic input validation in functions:

function calculateArea(width, height) {
    // Validate that inputs are numbers
    if (typeof width !== "number" || typeof height !== "number") {
        throw new Error("Both width and height must be numbers");
    }
    
    return width * height;
}

Let's Put It All Together

Let's see how variables, data types, and type conversion work together in a practical example:

// A simple shopping cart calculator
function calculateTotal(cart, taxRate, discount) {
    // Check if inputs are valid
    if (!Array.isArray(cart)) {
        throw new Error("Cart must be an array");
    }
    
    if (typeof taxRate !== "number") {
        console.warn("Tax rate is not a number, converting to number");
        taxRate = Number(taxRate);
        
        // Check if conversion was successful
        if (isNaN(taxRate)) {
            taxRate = 0;
            console.warn("Invalid tax rate provided, using 0");
        }
    }
    
    // Calculate subtotal
    let subtotal = 0;
    for (let i = 0; i < cart.length; i++) {
        const item = cart[i];
        
        // Check if item has valid price and quantity
        if (typeof item.price === "number" && typeof item.quantity === "number") {
            subtotal += item.price * item.quantity;
        } else {
            console.warn(`Skipping item ${item.name || "unknown"} due to invalid price or quantity`);
        }
    }
    
    // Apply discount
    let discountAmount = 0;
    if (discount) {
        if (typeof discount === "number") {
            // Discount is a fixed amount
            discountAmount = discount;
        } else if (typeof discount === "string" && discount.endsWith("%")) {
            // Discount is a percentage
            const percentage = Number(discount.slice(0, -1)) / 100;
            discountAmount = subtotal * percentage;
        }
    }
    
    // Calculate tax and final total
    const discountedSubtotal = subtotal - discountAmount;
    const taxAmount = discountedSubtotal * taxRate;
    const total = discountedSubtotal + taxAmount;
    
    // Return all calculated values
    return {
        subtotal: subtotal.toFixed(2),
        discount: discountAmount.toFixed(2),
        tax: taxAmount.toFixed(2),
        total: total.toFixed(2)
    };
}

// Example usage
const cart = [
    { name: "Laptop", price: 999.99, quantity: 1 },
    { name: "Mouse", price: 29.99, quantity: 2 },
    { name: "Keyboard", price: 79.99, quantity: 1 }
];

// Using different data types for inputs
const result1 = calculateTotal(cart, 0.08, 50);
console.log("Fixed discount:", result1);

const result2 = calculateTotal(cart, "0.08", "10%");
console.log("Percentage discount:", result2);

In the Real World: E-commerce Websites

The example above demonstrates a simplified version of what happens behind the scenes in e-commerce websites. When a user adds products to their cart, the application needs to:

  1. Store product information in appropriate data types
  2. Perform calculations with prices and quantities
  3. Apply discounts that might be fixed amounts or percentages
  4. Calculate taxes based on local rates
  5. Format the final numbers for display

All of these operations involve working with variables of different types and performing conversions between them as needed.

What is a REST API?

REST (Representational State Transfer) API is an architectural style for building web services that allows different applications to communicate using standard HTTP methods.

Key REST Principles

  • Stateless: Each request contains all information needed
  • Client-Server: Separation of concerns
  • Cacheable: Responses can be cached
  • Uniform Interface: Standard HTTP methods (GET, POST, PUT, DELETE)
  • Resource-Based: Data is treated as resources with URLs

WordPress REST API Overview

What is the WordPress REST API?

The WordPress REST API provides API endpoints for WordPress data types that allow developers to interact with sites remotely by sending and receiving JSON objects.

üí° Key Features:

  • Built into WordPress Core since version 4.7
  • Access WordPress data without loading the entire WordPress site
  • Create, read, update, and delete WordPress content
  • Extend with custom endpoints
  • Authentication and permission handling
  • JSON format for data exchange

Default WordPress REST API Endpoints

Core Endpoints

WordPress provides built-in endpoints for all standard content types:

Posts

GET /wp-json/wp/v2/posts

POST /wp-json/wp/v2/posts

PUT /wp-json/wp/v2/posts/{id}

DELETE /wp-json/wp/v2/posts/{id}

Pages

GET /wp-json/wp/v2/pages

Users

GET /wp-json/wp/v2/users

Categories

GET /wp-json/wp/v2/categories

Media

GET /wp-json/wp/v2/media

Comments

GET /wp-json/wp/v2/comments

Making Your First API Request

Simple GET Request

// Using Fetch API
fetch('https://yoursite.com/wp-json/wp/v2/posts')
    .then(response => response.json())
    .then(posts => {
        console.log('Posts:', posts);
        posts.forEach(post => {
            console.log(post.title.rendered);
        });
    })
    .catch(error => console.error('Error:', error));
// Using PHP
$response = wp_remote_get('https://yoursite.com/wp-json/wp/v2/posts');

if (!is_wp_error($response)) {
    $posts = json_decode(wp_remote_retrieve_body($response), true);
    
    foreach ($posts as $post) {
        echo $post['title']['rendered'];
    }
}
# Using cURL
curl https://yoursite.com/wp-json/wp/v2/posts

Tools for Working with REST API

  • Postman: API testing and documentation

  • Browser DevTools: Network tab for API inspection
  • WP-CLI: Command-line interface for WordPress
  • REST API Console: WordPress plugin for testing
  • JSONView: Browser extension for JSON formatting

REST API Best Practices

‚úÖ Do's

  • Use proper HTTP methods (GET for reading, POST for creating)
  • Implement authentication for sensitive operations
  • Version your custom endpoints
  • Return appropriate HTTP status codes
  • Validate and sanitize all input
  • Implement pagination for large datasets
  • Cache API responses when possible
  • Document your custom endpoints

‚ùå Don'ts

  • Don't expose sensitive data without authentication
  • Don't ignore rate limiting
  • Don't return excessive data (use field filtering)
  • Don't modify core endpoints directly
  • Don't forget CORS configuration for cross-origin requests
  • Don't skip error handling

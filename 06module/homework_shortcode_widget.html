Detailed Requirements

🎯 Core Requirements

  • Plugin Structure:
    • Proper plugin header with all required information
    • Organized file structure (separate files for shortcodes, widgets, admin, etc.)
    • Activation and deactivation hooks
    • Uninstall functionality to clean up data
  • Shortcodes (minimum 3):
    • Content Grid Shortcode - Display posts/pages in a grid layout
    • Team Members Shortcode - Show team member profiles
    • Testimonials Shortcode - Display customer testimonials
    • Each shortcode must have at least 5 customizable attributes
    • At least one shortcode must support nested content
  • Widgets (minimum 2):
    • Featured Content Widget - Highlight specific posts/pages
    • Social Proof Widget - Display statistics, testimonials, or social media feeds
    • Each widget must have a comprehensive settings form
    • Proper form validation and sanitization
  • Admin Interface:
    • Settings page under WordPress admin menu
    • Global plugin settings
    • Shortcode generator or documentation
    • Widget preview capability
  • Styling & Scripts:
    • Custom CSS file for frontend styling
    • Admin-specific CSS for backend
    • JavaScript for interactive features
    • Proper enqueuing of all assets

Starter Code Template

load_dependencies();
        $this->define_admin_hooks();
        $this->define_public_hooks();
        $this->register_shortcodes();
        $this->register_widgets();
    }
    
    /**
     * Load required files
     */
    private function load_dependencies() {
        // Load admin class
        require_once CSP_PLUGIN_DIR . 'admin/class-admin.php';
        
        // Load public class
        require_once CSP_PLUGIN_DIR . 'public/class-public.php';
        
        // Load shortcode classes
        require_once CSP_PLUGIN_DIR . 'shortcodes/class-content-grid.php';
        require_once CSP_PLUGIN_DIR . 'shortcodes/class-team-members.php';
        require_once CSP_PLUGIN_DIR . 'shortcodes/class-testimonials.php';
        
        // Load widget classes
        require_once CSP_PLUGIN_DIR . 'widgets/class-featured-content.php';
        require_once CSP_PLUGIN_DIR . 'widgets/class-social-proof.php';
    }
    
    /**
     * Register admin hooks
     */
    private function define_admin_hooks() {
        $admin = new CSP_Admin();
        
        add_action( 'admin_menu', array( $admin, 'add_admin_menu' ) );
        add_action( 'admin_enqueue_scripts', array( $admin, 'enqueue_admin_assets' ) );
        add_action( 'admin_init', array( $admin, 'register_settings' ) );
    }
    
    /**
     * Register public hooks
     */
    private function define_public_hooks() {
        $public = new CSP_Public();
        
        add_action( 'wp_enqueue_scripts', array( $public, 'enqueue_public_assets' ) );
    }
    
    /**
     * Register shortcodes
     */
    private function register_shortcodes() {
        $content_grid = new CSP_Content_Grid_Shortcode();
        $team_members = new CSP_Team_Members_Shortcode();
        $testimonials = new CSP_Testimonials_Shortcode();
        
        add_shortcode( 'csp_content_grid', array( $content_grid, 'render' ) );
        add_shortcode( 'csp_team_members', array( $team_members, 'render' ) );
        add_shortcode( 'csp_testimonials', array( $testimonials, 'render' ) );
    }
    
    /**
     * Register widgets
     */
    private function register_widgets() {
        add_action( 'widgets_init', function() {
            register_widget( 'CSP_Featured_Content_Widget' );
            register_widget( 'CSP_Social_Proof_Widget' );
        });
    }
}

// Activation hook
register_activation_hook( __FILE__, 'csp_activate' );
function csp_activate() {
    // Create database tables if needed
    // Set default options
    update_option( 'csp_version', CSP_VERSION );
    
    // Flush rewrite rules
    flush_rewrite_rules();
}

// Deactivation hook
register_deactivation_hook( __FILE__, 'csp_deactivate' );
function csp_deactivate() {
    // Clean up temporary data
    // Flush rewrite rules
    flush_rewrite_rules();
}

// Initialize plugin
add_action( 'plugins_loaded', function() {
    Content_Showcase_Pro::get_instance();
});

Tips for Success

💡 Development Tips

  • Start with one shortcode and get it working perfectly before moving to others
  • Use WordPress debugging tools (WP_DEBUG, Query Monitor plugin)
  • Test each feature thoroughly before moving to the next
  • Keep your code DRY (Don't Repeat Yourself) - create reusable functions
  • Use WordPress functions whenever possible instead of raw PHP
  • Comment your code to explain complex logic

⚠️ Common Pitfalls to Avoid

  • Not escaping output - always use esc_html(), esc_attr(), etc.
  • Not validating/sanitizing input data
  • Hardcoding paths - use plugin_dir_path() and plugin_dir_url()
  • Not checking if functions exist before defining them
  • Forgetting to enqueue scripts and styles properly
  • Not testing with different themes

Frequently Asked Questions

Q: Can I use a plugin boilerplate?

A: Yes, you can use a boilerplate like the WordPress Plugin Boilerplate, but make sure you understand the code and customize it appropriately.

Q: Should I include third-party libraries?

A: You can include libraries if needed, but document them clearly and ensure they're properly licensed. Prefer WordPress built-in functions when possible.

Q: How should I handle images for team members or testimonials?

A: Use the WordPress Media Library. Allow users to select images through the media uploader for better integration.

Q: Do I need to make it compatible with page builders?

A: Not required, but ensuring your shortcodes work within page builders is good practice and worth bonus points.

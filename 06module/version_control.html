Git Setup for WordPress Plugins

Initial Repository Setup

# Initialize Git repository
git init

# Create .gitignore file
cat > .gitignore << EOF
# Dependencies
node_modules/
vendor/

# Build files
dist/
build/
*.zip

# Development
.DS_Store
Thumbs.db
*.log
.env
.vscode/
.idea/

# WordPress
wp-config.php
EOF

# Initial commit
git add .
git commit -m "Initial commit"

# Add remote repository
git remote add origin https://github.com/username/plugin-name.git
git push -u origin main

Branching Strategies

Git Flow for Plugins

# Main branches
main        # Production-ready code
develop     # Development branch

# Feature branches
git checkout -b feature/ajax-support
git checkout -b feature/custom-post-type

# Release branches
git checkout -b release/1.2.0

# Hotfix branches
git checkout -b hotfix/security-fix

# Merge feature to develop
git checkout develop
git merge feature/ajax-support

# Create release
git checkout -b release/1.2.0 develop
# Test and fix bugs
git checkout main
git merge --no-ff release/1.2.0
git tag -a v1.2.0 -m "Version 1.2.0"

Semantic Versioning

Version Numbering

MAJOR.MINOR.PATCH

  • MAJOR: Breaking changes
  • MINOR: New features, backward compatible
  • PATCH: Bug fixes
# Update version in plugin header
/**
 * Plugin Name: My Plugin
 * Version: 1.2.3
 */

# Create version tag
git tag -a v1.2.3 -m "Release version 1.2.3"
git push origin v1.2.3

# List tags
git tag -l

# Checkout specific version
git checkout tags/v1.2.0

Collaboration Workflow

Pull Request Process

# Fork and clone repository
git clone https://github.com/yourusername/plugin.git

# Add upstream remote
git remote add upstream https://github.com/original/plugin.git

# Create feature branch
git checkout -b feature/new-feature

# Make changes and commit
git add .
git commit -m "Add new feature"

# Push to your fork
git push origin feature/new-feature

# Keep fork updated
git fetch upstream
git checkout main
git merge upstream/main

Commit Best Practices

Writing Good Commit Messages

# Good commit messages
git commit -m "Add: User registration feature"
git commit -m "Fix: SQL injection vulnerability in search"
git commit -m "Update: Improve performance of query loop"
git commit -m "Remove: Deprecated functions"
git commit -m "Refactor: Restructure admin menu code"

# Commit message format
Type: Subject (50 chars max)

Body (optional, wrap at 72 chars)
- Explain what and why
- Reference issues

Footer (optional)
Fixes #123
Co-authored-by: Name

Deployment with Git

Automated Deployment

# GitHub Actions workflow
name: Deploy to WordPress.org
on:
  push:
    tags:
      - "v*"
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: WordPress Plugin Deploy
        uses: 10up/action-wordpress-plugin-deploy@stable
        env:
          SVN_PASSWORD: ${{ secrets.SVN_PASSWORD }}
          SVN_USERNAME: ${{ secrets.SVN_USERNAME }}

Managing Plugin Updates

Changelog Management

# CHANGELOG.md
## [1.2.0] - 2024-01-15
### Added
- New AJAX functionality
- Custom post type support

### Fixed
- Security vulnerability in form handling
- Performance issue with large datasets

### Changed
- Updated minimum PHP version to 7.4

### Removed
- Deprecated widget code

Best Practices

  • Commit early and often
  • Write meaningful commit messages
  • Use branches for features
  • Tag releases properly
  • Keep main branch stable
  • Review code before merging
  • Document changes in CHANGELOG

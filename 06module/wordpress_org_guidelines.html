Submission Process

Steps to Submit

  1. Create WordPress.org account
  2. Prepare plugin package
  3. Submit for review
    • Go to https://wordpress.org/plugins/developers/add/
    • Upload ZIP file
    • Wait for initial review (1-10 days)
  4. Address review feedback
  5. Get approval
  6. Set up SVN repository

Common Violations to Avoid

‚ùå Common Issues:
  • Including libraries already in WordPress
  • Calling external services without disclosure
  • Not escaping/sanitizing data properly
  • Using non-GPL compatible code
  • Keyword stuffing in readme
  • Not prefixing functions/classes

Security Requirements

Security Best Practices

// Required: Nonce verification
if ( ! wp_verify_nonce( $_POST['nonce'], 'my_action' ) ) {
    die( 'Security check' );
}

// Required: Capability checks
if ( ! current_user_can( 'manage_options' ) ) {
    wp_die( 'Unauthorized' );
}

// Required: Data sanitization
$input = sanitize_text_field( $_POST['input'] );

// Required: SQL prepared statements
$results = $wpdb->get_results(
    $wpdb->prepare(
        "SELECT * FROM {$wpdb->posts} WHERE ID = %d",
        $id
    )
);

// Required: Escape output
echo esc_html( $user_content );
echo esc_url( $url );
echo esc_attr( $attribute );

Semantic Versioning

Version Numbering System

MAJOR.MINOR.PATCH

  • MAJOR: Breaking changes
  • MINOR: New features (backward compatible)
  • PATCH: Bug fixes
// Plugin header version
/**
 * Plugin Name: My Awesome Plugin
 * Version: 1.2.3
 */

// Update version in multiple files
const version = '1.2.3';

// package.json
{
  "version": "1.2.3"
}

// readme.txt
Stable tag: 1.2.3

// Create Git tag
git tag -a v1.2.3 -m "Release version 1.2.3"
git push origin v1.2.3

Git Hooks for Automation

Pre-commit Hook

#!/bin/sh
# .git/hooks/pre-commit

# Run PHP CodeSniffer
./vendor/bin/phpcs --standard=WordPress .
if [ $? -ne 0 ]; then
    echo "Coding standards check failed. Please fix errors."
    exit 1
fi

# Run PHPUnit tests
./vendor/bin/phpunit
if [ $? -ne 0 ]; then
    echo "Tests failed. Please fix failing tests."
    exit 1
fi

# Check for debug code
if grep -r "var_dump\|print_r\|console.log" --include="*.php" --include="*.js" .; then
    echo "Debug code found. Please remove before committing."
    exit 1
fi

exit 0

Collaboration Workflow

Pull Request Process

# Fork and clone repository
git clone https://github.com/username/plugin.git
cd plugin

# Add upstream remote
git remote add upstream https://github.com/original/plugin.git

# Create feature branch
git checkout -b feature/my-feature

# Make changes and commit
git add .
git commit -m "feat: Add new feature"

# Push to your fork
git push origin feature/my-feature

# Create pull request on GitHub

# Keep fork updated
git fetch upstream
git checkout main
git merge upstream/main
git push origin main

Release Management

Creating a Release

#!/bin/bash
# release.sh

VERSION=$1

# Update version in files
sed -i "s/Version: .*/Version: $VERSION/" my-plugin.php
sed -i "s/Stable tag: .*/Stable tag: $VERSION/" readme.txt
sed -i "s/\"version\": \".*\"/\"version\": \"$VERSION\"/" package.json

# Commit version bump
git add .
git commit -m "chore: Bump version to $VERSION"

# Create and push tag
git tag -a "v$VERSION" -m "Release version $VERSION"
git push origin main
git push origin "v$VERSION"

# Create release branch
git checkout -b "release/$VERSION"
git push origin "release/$VERSION"

echo "Release $VERSION created successfully!"

Best Practices

  • Commit early and often
  • Write meaningful commit messages
  • Never commit sensitive data
  • Use .gitignore effectively
  • Tag releases consistently
  • Keep branches up to date
  • Review code before merging
  • Document your Git workflow

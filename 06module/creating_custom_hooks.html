Make your plugins extensible with custom hooks

Learn how to create hooks that other developers can use

Learning Objectives

  • Understand why custom hooks are important
  • Create custom action hooks with do_action()
  • Create custom filter hooks with apply_filters()
  • Design extensible plugin architecture
  • Document custom hooks properly
  • Pass multiple parameters to hooks
  • Follow naming conventions
  • Test custom hooks effectively

Why Create Custom Hooks?

Custom hooks make your plugins extensible, allowing other developers to modify or extend your plugin's functionality without changing its core code.

Benefits of Custom Hooks

  • Extensibility: Other plugins can modify your plugin's behavior
  • Maintainability: Core code remains untouched during customization
  • Compatibility: Updates don't break customizations
  • Community: Developers can create add-ons for your plugin
  • Flexibility: Users can customize without editing files

Creating Custom Action Hooks

Use do_action() to create points where other code can execute.

Basic Custom Action Hook

<?php
// In your plugin code, create a hook point
function my_plugin_process_data() {
    // Before processing
    do_action( 'my_plugin_before_process' );
    
    // Your plugin's core functionality
    $data = get_option( 'my_plugin_data' );
    $processed = process_data( $data );
    
    // After processing - pass data to hooks
    do_action( 'my_plugin_after_process', $processed );
    
    return $processed;
}

// Other developers can now hook into these points:
add_action( 'my_plugin_before_process', 'custom_before_process' );

function custom_before_process() {
    // Custom code runs before your plugin processes data
    error_log( 'About to process data' );
}

add_action( 'my_plugin_after_process', 'custom_after_process' );

function custom_after_process( $processed_data ) {
    // Custom code runs after processing
    // Can access the processed data
    wp_mail( 'admin@example.com', 'Data Processed', print_r( $processed_data, true ) );
}

Action Hooks with Multiple Parameters

<?php
class My_Plugin_Form {
    
    public function save_form( $form_id, $form_data ) {
        // Validate form
        $is_valid = $this->validate( $form_data );
        
        // Fire hook before saving
        do_action( 
            'my_plugin_before_save_form', 
            $form_id,           // Parameter 1
            $form_data,         // Parameter 2
            $is_valid           // Parameter 3
        );
        
        if ( $is_valid ) {
            // Save to database
            $result = $this->save_to_db( $form_id, $form_data );
            
            // Fire hook after saving
            do_action(
                'my_plugin_after_save_form',
                $form_id,
                $form_data,
                $result
            );
            
            return $result;
        }
        
        // Fire hook for invalid form
        do_action( 'my_plugin_form_validation_failed', $form_id, $form_data );
        
        return false;
    }
}

// Developers can hook with multiple parameters
add_action( 'my_plugin_after_save_form', 'log_form_submission', 10, 3 );

function log_form_submission( $form_id, $form_data, $result ) {
    if ( $result ) {
        $log_entry = sprintf(
            'Form %d submitted successfully at %s',
            $form_id,
            current_time( 'mysql' )
        );
        error_log( $log_entry );
    }
}

Creating Custom Filter Hooks

Use apply_filters() to create points where data can be modified.

Basic Custom Filter Hook

<?php
// In your plugin, create filterable data points
function my_plugin_get_settings() {
    $default_settings = array(
        'enable_feature' => true,
        'items_per_page' => 10,
        'theme' => 'default'
    );
    
    // Allow other plugins to modify default settings
    $settings = apply_filters( 'my_plugin_default_settings', $default_settings );
    
    return $settings;
}

// Other developers can modify your data:
add_filter( 'my_plugin_default_settings', 'customize_plugin_settings' );

function customize_plugin_settings( $settings ) {
    // Modify the settings
    $settings['items_per_page'] = 20;
    $settings['theme'] = 'dark';
    
    // Must return the modified value
    return $settings;
}

Complex Filter Examples

<?php
class My_Plugin_Display {
    
    public function render_item( $item ) {
        // Get template
        $template = $this->get_template( $item->type );
        
        // Allow filtering of template
        $template = apply_filters(
            'my_plugin_item_template',
            $template,
            $item->type,
            $item
        );
        
        // Process template variables
        $variables = array(
            'title' => $item->title,
            'content' => $item->content,
            'date' => $item->date
        );
        
        // Allow filtering of variables
        $variables = apply_filters(
            'my_plugin_template_variables',
            $variables,
            $item,
            $template
        );
        
        // Render HTML
        $html = $this->render_template( $template, $variables );
        
        // Allow final filtering of HTML
        $html = apply_filters(
            'my_plugin_item_html',
            $html,
            $item,
            $template,
            $variables
        );
        
        return $html;
    }
    
    public function get_allowed_types() {
        $types = array( 'post', 'page', 'product' );
        
        // Allow adding custom types
        $types = apply_filters( 'my_plugin_allowed_types', $types );
        
        return $types;
    }
}

// Developers can modify at multiple points
add_filter( 'my_plugin_template_variables', 'add_custom_variables', 10, 3 );

function add_custom_variables( $variables, $item, $template ) {
    // Add custom variables
    $variables['author'] = get_the_author_meta( 'display_name', $item->author_id );
    $variables['category'] = get_the_category( $item->ID );
    
    return $variables;
}

add_filter( 'my_plugin_allowed_types', 'add_custom_post_type' );

function add_custom_post_type( $types ) {
    $types[] = 'portfolio';
    $types[] = 'testimonial';
    return $types;
}

4

Hook Naming Conventions

Best Practices for Hook Names

  • Use plugin prefix: my_plugin_hook_name
  • Be descriptive: my_plugin_before_render_gallery
  • Use underscores: Not hyphens or camelCase
  • Indicate timing: before_, after_, pre_, post_
  • Group related hooks: Same prefix for related functionality
Good Names ‚úÖ Bad Names ‚ùå Why?
my_plugin_before_save before_save No prefix - conflicts possible
my_plugin_filter_content myPlugin-filterContent Wrong separator style
my_plugin_after_init mp_ai Too abbreviated
my_plugin_modify_query MYPLUGIN_MODIFY All caps inappropriate

Real-World Plugin Example

Complete Extensible Plugin Class

<?php
/**
 * Extensible Contact Form Plugin
 */
class Extensible_Contact_Form {
    
    private $fields = array();
    private $settings = array();
    
    public function __construct() {
        $this->init();
    }
    
    private function init() {
        // Fire action when plugin initializes
        do_action( 'ecf_init', $this );
        
        // Load settings with filter
        $default_settings = array(
            'recipient' => get_option( 'admin_email' ),
            'subject' => 'Contact Form Submission',
            'success_message' => 'Thank you for contacting us!'
        );
        
        $this->settings = apply_filters( 'ecf_default_settings', $default_settings );
        
        // Set up default fields with filter
        $this->setup_fields();
    }
    
    private function setup_fields() {
        $default_fields = array(
            'name' => array(
                'type' => 'text',
                'label' => 'Name',
                'required' => true
            ),
            'email' => array(
                'type' => 'email',
                'label' => 'Email',
                'required' => true
            ),
            'message' => array(
                'type' => 'textarea',
                'label' => 'Message',
                'required' => true
            )
        );
        
        // Allow modifying fields
        $this->fields = apply_filters( 'ecf_form_fields', $default_fields );
    }
    
    public function render_form() {
        // Before form render
        do_action( 'ecf_before_form_render', $this->fields );
        
        $form_html = '
';
        
        // Allow filtering form attributes
        $form_attributes = apply_filters( 'ecf_form_attributes', array(
            'class' => 'ecf-form',
            'id' => 'contact-form'
        ) );
        
        foreach ( $this->fields as $field_name => $field ) {
            // Allow modifying individual fields
            $field = apply_filters( 'ecf_render_field', $field, $field_name );
            
            $form_html .= $this->render_field( $field_name, $field );
        }
        
        // Add submit button with filter
        $submit_text = apply_filters( 'ecf_submit_button_text', 'Send Message' );
        $form_html .= '';
        
        $form_html .= '
';
        
        // Filter final form HTML
        $form_html = apply_filters( 'ecf_form_html', $form_html, $this->fields );
        
        // After form render
        do_action( 'ecf_after_form_render', $form_html );
        
        return $form_html;
    }
    
    public function process_submission( $data ) {
        // Before validation
        do_action( 'ecf_before_validation', $data );
        
        // Validate with filter
        $is_valid = apply_filters( 'ecf_validate_submission', true, $data );
        
        if ( ! $is_valid ) {
            do_action( 'ecf_validation_failed', $data );
            return false;
        }
        
        // Before sending
        do_action( 'ecf_before_send_email', $data );
        
        // Prepare email
        $email_data = apply_filters( 'ecf_email_data', array(
            'to' => $this->settings['recipient'],
            'subject' => $this->settings['subject'],
            'message' => $this->format_message( $data ),
            'headers' => array()
        ), $data );
        
        // Send email
        $sent = wp_mail(
            $email_data['to'],
            $email_data['subject'],
            $email_data['message'],
            $email_data['headers']
        );
        
        if ( $sent ) {
            // After successful send
            do_action( 'ecf_email_sent', $data, $email_data );
        } else {
            // On send failure
            do_action( 'ecf_email_failed', $data, $email_data );
        }
        
        return $sent;
    }
    
    private function format_message( $data ) {
        $message = '';
        
        foreach ( $data as $field => $value ) {
            $message .= ucfirst( $field ) . ': ' . $value . "\n";
        }
        
        // Allow filtering the message format
        return apply_filters( 'ecf_format_message', $message, $data );
    }
}

// Example of extending the plugin
add_filter( 'ecf_form_fields', 'add_phone_field' );

function add_phone_field( $fields ) {
    $fields['phone'] = array(
        'type' => 'tel',
        'label' => 'Phone Number',
        'required' => false
    );
    return $fields;
}

add_action( 'ecf_email_sent', 'log_form_submission', 10, 2 );

function log_form_submission( $data, $email_data ) {
    // Log successful submissions
    error_log( 'Form submitted: ' . print_r( $data, true ) );
}

Documenting Your Hooks

Proper Hook Documentation

<?php
/**
 * Fires before the widget content is rendered.
 *
 * @since 1.0.0
 *
 * @param array  $instance The widget instance settings.
 * @param string $widget_id The widget ID.
 * @param array  $args The widget arguments.
 */
do_action( 'my_plugin_before_widget_render', $instance, $widget_id, $args );

/**
 * Filters the widget output HTML.
 *
 * @since 1.0.0
 *
 * @param string $html The widget HTML output.
 * @param array  $instance The widget instance settings.
 * @param string $widget_id The widget ID.
 * 
 * @return string Modified widget HTML.
 */
$html = apply_filters( 'my_plugin_widget_html', $html, $instance, $widget_id );

Testing Your Custom Hooks

Hook Testing Example

<?php
// Test file for your custom hooks
class Test_My_Plugin_Hooks {
    
    public function run_tests() {
        // Test action hook
        add_action( 'my_plugin_test_action', array( $this, 'test_action_callback' ), 10, 2 );
        
        // Fire the action
        do_action( 'my_plugin_test_action', 'param1', 'param2' );
        
        // Test filter hook
        add_filter( 'my_plugin_test_filter', array( $this, 'test_filter_callback' ) );
        
        // Apply the filter
        $original = 'original value';
        $filtered = apply_filters( 'my_plugin_test_filter', $original );
        
        // Verify filter worked
        if ( $filtered !== $original ) {
            echo 'Filter test passed!';
        }
    }
    
    public function test_action_callback( $param1, $param2 ) {
        echo "Action fired with: $param1, $param2";
    }
    
    public function test_filter_callback( $value ) {
        return $value . ' - modified';
    }
}

Custom Hook Best Practices

  • Always prefix: Use your plugin slug as prefix
  • Be consistent: Use same naming pattern throughout
  • Document thoroughly: Explain when hooks fire and parameters
  • Pass useful data: Include relevant context in parameters
  • Check for existence: Use did_action() and has_filter()
  • Version your hooks: Document when hooks were added
  • Provide examples: Show how to use your hooks
  • Consider performance: Don't create too many hooks
Don't change hook parameters in minor versions - it breaks backward compatibility!
Create a developer documentation page listing all your custom hooks with examples.

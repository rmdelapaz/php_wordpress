Understanding the two fundamental types of WordPress hooks

Learn when to use actions and when to use filters in your plugins

Learning Objectives

  • Understand the fundamental difference between actions and filters
  • Learn when to use action hooks vs filter hooks
  • Master the syntax for both hook types
  • Understand return values and parameters
  • Apply best practices for each hook type
  • Debug hooks effectively
  • Chain multiple hooks together
  • Handle hook errors gracefully

The WordPress Hook System

WordPress hooks are the foundation of plugin development. They allow you to "hook into" WordPress at specific points and either execute code (actions) or modify data (filters).

Hook System Overview

WordPress Core ‚Üí Action Hook ‚Üí Execute Code


WordPress Core ‚Üí Filter Hook ‚Üí Modify Data ‚Üí Return Data

Action Hooks

üé¨ Actions DO SOMETHING

Action hooks allow you to execute code at specific points in WordPress execution. They don't return values - they simply run code.

Key Characteristics:

  • Execute functions at specific points
  • Don't return values
  • Can't modify data directly
  • Used for side effects (database updates, sending emails, etc.)
  • Multiple functions can hook to the same action

Action Hook Syntax

<?php
// Creating an action hook (in WordPress core or your plugin)
do_action( 'my_custom_action', $arg1, $arg2 );

// Hooking into an action
add_action( 'hook_name', 'callback_function', $priority, $accepted_args );

// Example: Basic action hook
add_action( 'init', 'my_init_function' );

function my_init_function() {
    // This code runs when WordPress initializes
    // No return value needed
    register_post_type( 'product', $args );
}

// Example: Action with parameters
add_action( 'save_post', 'my_save_post_function', 10, 3 );

function my_save_post_function( $post_id, $post, $update ) {
    // Do something when a post is saved
    if ( $update ) {
        // Post was updated
        error_log( "Post $post_id was updated" );
    }
}

// Example: Multiple actions on same hook
add_action( 'wp_footer', 'add_analytics_code' );
add_action( 'wp_footer', 'add_custom_javascript' );
add_action( 'wp_footer', 'add_footer_message' );

function add_analytics_code() {
    echo '<!-- Google Analytics Code -->';
}

function add_custom_javascript() {
    echo '<script>console.log("Footer loaded");</script>';
}

function add_footer_message() {
    echo '<!-- End of page -->';
}

Filter Hooks

üîÑ Filters MODIFY DATA

Filter hooks allow you to modify data before it's used. They receive data, modify it, and must return it.

Key Characteristics:

  • Receive data as input
  • Must return modified (or unmodified) data
  • Can chain multiple filters
  • Used to modify content, settings, queries
  • Order matters when multiple filters are applied

Filter Hook Syntax

<?php
// Creating a filter hook (in WordPress core or your plugin)
$value = apply_filters( 'my_custom_filter', $value, $arg1, $arg2 );

// Hooking into a filter
add_filter( 'hook_name', 'callback_function', $priority, $accepted_args );

// Example: Basic filter hook
add_filter( 'the_content', 'my_content_filter' );

function my_content_filter( $content ) {
    // Modify the content
    $content .= '<p>Thanks for reading!</p>';
    
    // MUST return the content
    return $content;
}

// Example: Filter with multiple parameters
add_filter( 'the_title', 'my_title_filter', 10, 2 );

function my_title_filter( $title, $post_id ) {
    // Add emoji to certain post titles
    if ( get_post_type( $post_id ) === 'product' ) {
        $title = 'üõçÔ∏è ' . $title;
    }
    
    return $title;
}

// Example: Chaining filters
add_filter( 'the_content', 'filter_one', 10 );
add_filter( 'the_content', 'filter_two', 20 );
add_filter( 'the_content', 'filter_three', 30 );

function filter_one( $content ) {
    return $content . '<!-- Filter 1 -->';
}

function filter_two( $content ) {
    return $content . '<!-- Filter 2 -->';
}

function filter_three( $content ) {
    return $content . '<!-- Filter 3 -->';
}
// Result: Original content + Filter 1 + Filter 2 + Filter 3
Always return a value from filter functions! Forgetting to return will result in empty/null data.

Detailed Comparison

Aspect Action Hooks Filter Hooks
Purpose Execute code at specific points Modify data before it's used
Return Value None (void) Required (modified data)
WordPress Function do_action() apply_filters()
Hook Function add_action() add_filter()
Common Use Cases Send emails, save data, enqueue scripts Modify content, change URLs, alter queries
Can Break Site If Wrong Less likely More likely (if no return)
Execution Just runs the function Replaces value with return
Debugging Can use echo/var_dump Must use error_log

Common Use Cases

Action Hook Use Cases

  • Initialize custom post types
  • Enqueue scripts and styles
  • Send email notifications
  • Log user activity
  • Create database entries
  • Clear caches
  • Schedule cron jobs
  • Redirect users
  • Display admin notices
  • Register widgets

Filter Hook Use Cases

  • Modify post content
  • Change excerpt length
  • Alter query parameters
  • Customize menu items
  • Modify user capabilities
  • Change email content
  • Filter search results
  • Modify URLs and links
  • Customize login messages
  • Alter image sizes

Real-World Examples

Action Example: Send Email on Post Publish

<?php
add_action( 'transition_post_status', 'notify_on_publish', 10, 3 );

function notify_on_publish( $new_status, $old_status, $post ) {
    // Check if post is being published for the first time
    if ( $old_status !== 'publish' && $new_status === 'publish' ) {
        
        // Get admin email
        $admin_email = get_option( 'admin_email' );
        
        // Prepare email
        $subject = 'New Post Published: ' . $post->post_title;
        $message = sprintf(
            'A new post has been published on your site.%sTitle: %s%sAuthor: %s%sView post: %s',
            "\n\n",
            $post->post_title,
            "\n",
            get_the_author_meta( 'display_name', $post->post_author ),
            "\n",
            get_permalink( $post->ID )
        );
        
        // Send email (action - no return needed)
        wp_mail( $admin_email, $subject, $message );
        
        // Log the action
        error_log( "Email sent for post ID: {$post->ID}" );
    }
}

Filter Example: Add Reading Time to Content

<?php
add_filter( 'the_content', 'add_reading_time', 10 );

function add_reading_time( $content ) {
    // Only on single posts
    if ( ! is_single() ) {
        return $content;
    }
    
    // Calculate reading time
    $word_count = str_word_count( strip_tags( $content ) );
    $reading_time = ceil( $word_count / 200 ); // 200 words per minute
    
    // Create reading time HTML
    $reading_time_html = sprintf(
        '<div class="reading-time">‚è±Ô∏è Estimated reading time: %d minute%s</div>',
        $reading_time,
        $reading_time === 1 ? '' : 's'
    );
    
    // Add to beginning of content
    $content = $reading_time_html . $content;
    
    // MUST return the modified content
    return $content;
}

Complex Example: Using Both Actions and Filters Together

<?php
// Plugin that tracks and displays post views

// ACTION: Track post views
add_action( 'wp_head', 'track_post_views' );

function track_post_views() {
    if ( is_single() ) {
        $post_id = get_the_ID();
        $views = get_post_meta( $post_id, '_post_views', true );
        $views = $views ? $views + 1 : 1;
        update_post_meta( $post_id, '_post_views', $views );
        
        // This is an action - no return needed
    }
}

// FILTER: Display view count in content
add_filter( 'the_content', 'display_post_views', 20 );

function display_post_views( $content ) {
    if ( is_single() ) {
        $post_id = get_the_ID();
        $views = get_post_meta( $post_id, '_post_views', true );
        $views = $views ? $views : 0;
        
        $view_html = sprintf(
            '<div class="post-views">üëÅÔ∏è This post has been viewed %d time%s</div>',
            $views,
            $views === 1 ? '' : 's'
        );
        
        // Add view count after content
        $content = $content . $view_html;
    }
    
    // This is a filter - MUST return content
    return $content;
}

// ACTION: Add custom column to admin
add_action( 'manage_posts_custom_column', 'display_views_column', 10, 2 );

function display_views_column( $column, $post_id ) {
    if ( $column === 'post_views' ) {
        $views = get_post_meta( $post_id, '_post_views', true );
        echo $views ? $views : '0';
        // Action - no return needed
    }
}

// FILTER: Add column header
add_filter( 'manage_posts_columns', 'add_views_column' );

function add_views_column( $columns ) {
    $columns['post_views'] = 'Views';
    // Filter - MUST return columns
    return $columns;
}

Common Mistakes to Avoid

‚ùå Wrong: Using echo in a filter

// WRONG - Don't echo in filters
add_filter( 'the_title', 'bad_title_filter' );

function bad_title_filter( $title ) {
    echo 'Modified: ';  // This will output in wrong place!
    return $title;
}

‚ùå Wrong: Forgetting to return in a filter

// WRONG - Must return value in filters
add_filter( 'the_content', 'bad_content_filter' );

function bad_content_filter( $content ) {
    $content .= '<p>Additional content</p>';
    // Forgot to return! Content will be empty!
}

‚úÖ Correct: Proper filter usage

// CORRECT - Always return in filters
add_filter( 'the_content', 'good_content_filter' );

function good_content_filter( $content ) {
    $content .= '<p>Additional content</p>';
    return $content;  // Always return!
}

Best Practices

  • Always return in filters: Even if unchanged, return the original value
  • Never echo in filters: Use return to output modified content
  • Check context: Use conditional tags (is_single(), is_admin(), etc.)
  • Unique function names: Prefix with your plugin slug
  • Document parameters: Comment what parameters your function expects
  • Set appropriate priority: Default is 10, lower runs first
  • Declare accepted args: Specify how many parameters you need
  • Test thoroughly: Check if hooks exist before using
Use has_action() and has_filter() to check if hooks are already registered before adding your own.

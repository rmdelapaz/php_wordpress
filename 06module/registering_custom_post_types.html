The register_post_type() Function

The register_post_type() function is the cornerstone of creating custom post types in WordPress.

register_post_type( string $post_type, array|string $args = array() )

Function Parameters

  • $post_type (string) - The name of the post type. Max 20 characters, lowercase, no spaces.
  • $args (array) - An array of arguments that define the post type's behavior and appearance.

Simple Registration Example

function create_book_post_type() {
    register_post_type( 'book',
        array(
            'labels' => array(
                'name' => __( 'Books' ),
                'singular_name' => __( 'Book' )
            ),
            'public' => true,
            'has_archive' => true,
            'rewrite' => array('slug' => 'books'),
            'show_in_rest' => true,
        )
    );
}
add_action( 'init', 'create_book_post_type' );
Important: Custom post types must be registered on the init action hook. Registering them too early or too late can cause issues.

Visibility Parameters

Parameter Type Default Description
public bool false Meta argument controlling multiple visibility settings
publicly_queryable bool value of public Whether queries for this post type can be performed from the front-end
show_ui bool value of public Whether to generate a default UI in admin
show_in_menu bool/string value of show_ui Where to show in admin menu. Can be bool or parent menu slug
show_in_nav_menus bool value of public Whether post type is available in navigation menus
show_in_admin_bar bool value of show_in_menu Whether to show in admin bar "New" menu
exclude_from_search bool opposite of public Whether to exclude from front-end search results

Feature Support

'supports' => array(
    'title',           // Post title
    'editor',          // Post content editor
    'author',          // Author metabox
    'thumbnail',       // Featured image
    'excerpt',         // Post excerpt
    'trackbacks',      // Trackback/pingback support
    'custom-fields',   // Custom fields metabox
    'comments',        // Comments
    'revisions',       // Post revisions
    'page-attributes', // Page attributes (menu order, parent)
    'post-formats',    // Post formats
),
Tip: You can also use add_post_type_support() and remove_post_type_support() to modify features after registration.

Admin UI Settings

Parameter Type Description
menu_position int Position in admin menu (5-100)
menu_icon string Menu icon (dashicon class, URL, or base64 SVG)
hierarchical bool Whether post type is hierarchical (like pages)
has_archive bool/string Enable archives (can specify custom slug)
can_export bool Can be exported via Tools → Export

Menu Position Values

  • 5 - below Posts
  • 10 - below Media
  • 15 - below Links
  • 20 - below Pages
  • 25 - below Comments
  • 60 - below first separator
  • 65 - below Plugins
  • 70 - below Users
  • 75 - below Tools
  • 80 - below Settings

Capabilities and Permissions

// Simple capability mapping
'capability_type' => 'book',

// Or detailed capability mapping
'capabilities' => array(
    'edit_post'          => 'edit_book',
    'read_post'          => 'read_book',
    'delete_post'        => 'delete_book',
    'edit_posts'         => 'edit_books',
    'edit_others_posts'  => 'edit_others_books',
    'publish_posts'      => 'publish_books',
    'read_private_posts' => 'read_private_books',
    'create_posts'       => 'create_books',
),

// Map meta capabilities
'map_meta_cap' => true,
Note: When using custom capabilities, you need to add them to user roles using add_cap().

REST API and URL Settings

// REST API Settings
'show_in_rest' => true,                    // Enable in REST API
'rest_base' => 'books',                    // REST API base slug
'rest_controller_class' => 'WP_REST_Posts_Controller', // REST controller

// URL Rewrite Settings
'rewrite' => array(
    'slug' => 'books',                     // Custom slug
    'with_front' => true,                  // Prepend with front base
    'feeds' => true,                        // Generate feed permalinks
    'pages' => true,                        // Generate paginated permalinks
),

// Query Variable
'query_var' => true,                       // Can be string to customize
Tip: Setting show_in_rest to true enables Gutenberg block editor support!

Complete Registration Example

Full-Featured Event CPT

Here's a comprehensive example of registering an "Events" custom post type with all best practices:

/**
 * Plugin Name: Event Manager
 * Description: Manages events with a custom post type
 * Version: 1.0.0
 */

    exit;
}

class Event_Post_Type {
    
    /**
     * Constructor
     */
    public function __construct() {
        add_action( 'init', array( $this, 'register_event_post_type' ) );
        add_action( 'init', array( $this, 'register_event_taxonomies' ) );
        add_filter( 'enter_title_here', array( $this, 'change_title_placeholder' ) );
        add_action( 'admin_head', array( $this, 'add_custom_admin_styles' ) );
        
        // Flush rewrite rules on activation
        register_activation_hook( __FILE__, array( $this, 'flush_rewrites' ) );
    }
    
    /**
     * Register the Event post type
     */
    public function register_event_post_type() {
        
        $labels = array(
            'name'                  => _x( 'Events', 'Post type general name', 'event-manager' ),
            'singular_name'         => _x( 'Event', 'Post type singular name', 'event-manager' ),
            'menu_name'             => _x( 'Events', 'Admin Menu text', 'event-manager' ),
            'name_admin_bar'        => _x( 'Event', 'Add New on Toolbar', 'event-manager' ),
            'add_new'               => __( 'Add New', 'event-manager' ),
            'add_new_item'          => __( 'Add New Event', 'event-manager' ),
            'new_item'              => __( 'New Event', 'event-manager' ),
            'edit_item'             => __( 'Edit Event', 'event-manager' ),
            'view_item'             => __( 'View Event', 'event-manager' ),
            'all_items'             => __( 'All Events', 'event-manager' ),
            'search_items'          => __( 'Search Events', 'event-manager' ),
            'parent_item_colon'     => __( 'Parent Events:', 'event-manager' ),
            'not_found'             => __( 'No events found.', 'event-manager' ),
            'not_found_in_trash'    => __( 'No events found in Trash.', 'event-manager' ),
            'featured_image'        => _x( 'Event Featured Image', 'Overrides the "Featured Image" phrase', 'event-manager' ),
            'set_featured_image'    => _x( 'Set featured image', 'Overrides the "Set featured image" phrase', 'event-manager' ),
            'remove_featured_image' => _x( 'Remove featured image', 'Overrides the "Remove featured image" phrase', 'event-manager' ),
            'use_featured_image'    => _x( 'Use as featured image', 'Overrides the "Use as featured image" phrase', 'event-manager' ),
            'archives'              => _x( 'Event archives', 'The post type archive label', 'event-manager' ),
            'insert_into_item'      => _x( 'Insert into event', 'Overrides the "Insert into post" phrase', 'event-manager' ),
            'uploaded_to_this_item' => _x( 'Uploaded to this event', 'Overrides the "Uploaded to this post" phrase', 'event-manager' ),
            'filter_items_list'     => _x( 'Filter events list', 'Screen reader text', 'event-manager' ),
            'items_list_navigation' => _x( 'Events list navigation', 'Screen reader text', 'event-manager' ),
            'items_list'            => _x( 'Events list', 'Screen reader text', 'event-manager' ),
        );
        
        $args = array(
            'labels'                => $labels,
            'description'           => __( 'Events and event management', 'event-manager' ),
            'public'                => true,
            'publicly_queryable'    => true,
            'show_ui'               => true,
            'show_in_menu'          => true,
            'query_var'             => true,
            'rewrite'               => array( 
                'slug' => 'events',
                'with_front' => false,
                'feeds' => true,
                'pages' => true
            ),
            'capability_type'       => 'post',
            'has_archive'           => 'events',
            'hierarchical'          => false,
            'menu_position'         => 20,
            'menu_icon'             => 'dashicons-calendar-alt',
            'supports'              => array( 
                'title', 
                'editor', 
                'author',

            'show_in_rest'          => true,

            'taxonomies'            => array( 'event_category', 'event_tag' ),

    }
    
    /**
     * Register Event taxonomies
     */
    public function register_event_taxonomies() {
        // We'll cover this in the next lesson!
        // Event Categories (hierarchical like categories)
        // Event Tags (non-hierarchical like tags)
        // Event Venues (custom taxonomy)
    }
    
    /**
     * Change title placeholder text
     */
    public function change_title_placeholder( $title ) {
        $screen = get_current_screen();
        
        if ( 'event' == $screen->post_type ) {
            $title = 'Enter event name here';
        }
        
        return $title;
    }
    
    /**
     * Add custom admin styles for our CPT
     */

Step-by-Step Implementation Guide

Step 1: Plan Your Post Type

Before writing code, decide on:

  • Post type name (singular, lowercase, no spaces)
  • Features needed (editor, thumbnail, etc.)
  • URL structure
  • Admin menu position and icon
  • Whether it needs REST API support

Step 2: Create the Registration Function

function register_my_cpt() {
    $args = array(
        // Your arguments here
    );
    register_post_type( 'my_cpt', $args );
}
add_action( 'init', 'register_my_cpt' );

Step 3: Add Labels

Create comprehensive labels for better UX:

$labels = array(
    'name' => __( 'Items' ),
    'singular_name' => __( 'Item' ),
    // Add all other labels
);

Step 4: Configure Arguments

Set up the behavior and features:

$args = array(
    'labels' => $labels,
    'public' => true,
    'has_archive' => true,
    'show_in_rest' => true,
    // Other arguments
);

Step 5: Flush Rewrite Rules

Important for permalinks to work:

// On plugin activation
register_activation_hook( __FILE__, 'my_rewrite_flush' );
function my_rewrite_flush() {
    register_my_cpt();
    flush_rewrite_rules();
}

Step 6: Test Everything

  • Check admin menu appearance
  • Create a test post
  • Verify permalinks work
  • Test REST API endpoints
  • Check front-end display

Common Issues and Solutions

Issue: 404 Error on CPT Pages

Cause: Permalinks need to be refreshed.

Solution: Go to Settings → Permalinks and click "Save Changes" or flush rewrite rules programmatically.

Issue: CPT Not Showing in Admin Menu

Cause: show_ui or show_in_menu is false.

Solution: Ensure both are set to true or check public parameter.

Issue: Gutenberg Not Available

Cause: REST API support not enabled.

Solution: Set 'show_in_rest' => true in your arguments.

Issue: Custom Capabilities Not Working

Cause: Capabilities not added to user roles.

Solution: Add capabilities to roles using $role->add_cap().

Best Practices

✅ Do's

  • Always use prefixes to avoid naming conflicts
  • Make your post types translatable with text domains
  • Flush rewrite rules on activation/deactivation only
  • Use object-oriented approach for complex implementations
  • Enable REST API for modern WordPress development
  • Provide comprehensive labels for better UX
  • Document your custom post types

❌ Don'ts

  • Don't flush rewrite rules on every page load
  • Don't use reserved post type names
  • Don't register CPTs before 'init' hook
  • Don't use spaces or capital letters in post type names
  • Don't exceed 20 characters for post type names
  • Don't forget to handle uninstall cleanup

Understanding Template Hierarchy

WordPress uses a specific hierarchy to determine which template file to use when displaying content. Understanding this hierarchy is crucial for creating custom post type templates.

🔍 How WordPress Chooses Templates

WordPress searches for template files in this order and uses the first one it finds:

For Single CPT Posts:

1 single-{post_type}-{slug}.php Most specific - for individual posts
2 single-{post_type}.php For all posts of this type
3 single.php Generic single post template
4 singular.php For any singular content
5 index.php Ultimate fallback

For CPT Archives:

1 archive-{post_type}.php Specific CPT archive
2 archive.php Generic archive template
3 index.php Ultimate fallback

Conditional Tags for CPTs

Essential Conditional Tags

Check Post Type is_singular('product')
CPT Archive is_post_type_archive('product')
Get Post Type get_post_type()
Taxonomy Archive is_tax('product_category') 
// Use conditional tags in templates
if ( is_singular( 'product' ) ) {
    // Single product page
    get_template_part( 'template-parts/single', 'product' );
    
} elseif ( is_post_type_archive( 'product' ) ) {
    // Product archive page
    get_template_part( 'template-parts/archive', 'product' );
    
} elseif ( is_tax( 'product_category' ) ) {
    // Product category taxonomy page
    $term = get_queried_object();
    get_template_part( 'template-parts/taxonomy', 'product-category' );
}

Loading Templates from Plugins

Plugin Template Loader

class CPT_Template_Loader {
    
    public function __construct() {
        add_filter( 'template_include', array( $this, 'template_loader' ) );
    }
    
    public function template_loader( $template ) {
        if ( is_singular( 'product' ) ) {
            $template = $this->get_template( 'single-product.php' );
        } elseif ( is_post_type_archive( 'product' ) ) {
            $template = $this->get_template( 'archive-product.php' );
        } elseif ( is_tax( 'product_category' ) ) {
            $template = $this->get_template( 'taxonomy-product_category.php' );
        }
        
        return $template;
    }
    
    private function get_template( $template_name ) {
        // Check if template exists in theme
        $theme_file = locate_template( array( 'my-plugin/' . $template_name ) );
        
        if ( $theme_file ) {
            return $theme_file;
        }
        
        // Use plugin template
        $plugin_file = plugin_dir_path( __FILE__ ) . 'templates/' . $template_name;
        
        if ( file_exists( $plugin_file ) ) {
            return $plugin_file;
        }
        
        return $template;
    }
}

new CPT_Template_Loader();

Best Practices for CPT Templates

✅ Template Best Practices

  • Use semantic HTML5 elements
  • Include proper schema.org markup
  • Make templates responsive by default
  • Use template parts for reusable components
  • Follow WordPress coding standards
  • Include proper hooks and filters
  • Test with various themes

❌ Common Mistakes

  • Hardcoding paths instead of using WordPress functions
  • Not checking if functions exist before using
  • Forgetting to reset post data
  • Not escaping output properly
  • Creating templates that break with pagination

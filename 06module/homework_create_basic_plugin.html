Create a Basic WordPress Plugin

Apply everything you've learned to build a functional plugin

Assignment Overview

In this homework assignment, you will create a complete WordPress plugin from scratch. Your plugin will be called "Student Portfolio" and will allow users to showcase their projects and skills on their WordPress site.

This assignment will test your understanding of:

  • Plugin architecture and structure
  • WordPress hooks and filters
  • Database operations
  • Admin interface creation
  • Shortcode implementation
  • Security best practices
  • Proper file organization

üìã Project Requirements

Core Features (Required)

Create proper plugin file structure with main file, includes folder, assets folder, and proper header information
Register a custom post type called "portfolio_item" with title, editor, thumbnail, and custom fields support
Create a custom taxonomy called "portfolio_category" for organizing portfolio items
Create an admin settings page under a custom menu with at least 3 configurable options
Create a [portfolio] shortcode that displays portfolio items with parameters for category and count
Create a custom database table to store portfolio view statistics
Implement proper activation hook to create database table and deactivation hook for cleanup
Properly enqueue CSS and JavaScript files for both admin and frontend
Use nonces for forms, validate and sanitize all inputs, escape all outputs
Create uninstall.php to properly clean up all plugin data when deleted

Implementation Hints

Start with the basic plugin structure and header. Make sure WordPress recognizes your plugin before adding functionality.
Use the WordPress Plugin Boilerplate as a reference, but write your own code to ensure understanding.
Test activation and deactivation hooks early. Use error_log() to debug if needed.
For the custom post type, use the register_post_type() function with appropriate arguments for labels and supports.
When creating the database table, use the dbDelta() function for proper table creation and updates.

Common Mistakes to Avoid

Don't forget to prefix all your functions, classes, and database tables to avoid conflicts.
Always check if functions exist before defining them: if ( ! function_exists( 'function_name' ) )
Don't echo directly in shortcodes - always return the output.
Remember to use wp_die() with proper error messages in activation if requirements aren't met.
Don't delete user data in deactivation hook - only in uninstall.

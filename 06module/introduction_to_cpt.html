Extend WordPress beyond posts and pages

Learn to create custom content types for any purpose

What Are Custom Post Types?

Custom Post Types (CPTs) are content types that you can create in WordPress beyond the default posts and pages. They allow you to organize and display different types of content with their own unique properties, admin interfaces, and templates.

WordPress Content Hierarchy

WordPress Content
Built-in Post Types
  • Posts (blog entries)
  • Pages (static content)
  • Attachments (media)
  • Revisions (version history)
  • Navigation Menu Items
Custom Post Types
  • Products
  • Portfolio Items
  • Testimonials
  • Events
  • Team Members
  • Any custom content...

WordPress Built-in Post Types

post page attachment revision nav_menu_item custom_css customize_changeset wp_block wp_template wp_template_part

Common Use Cases

E-Commerce: Products

// Product custom post type for an online store
register_post_type( 'product', array(
    'labels' => array(
        'name' => 'Products',
        'singular_name' => 'Product',
    ),
    'public' => true,
    'has_archive' => true,
    'supports' => array( 'title', 'editor', 'thumbnail', 'excerpt', 'custom-fields' ),
    'menu_icon' => 'dashicons-cart',
));

Portfolio Website: Projects

// Portfolio items for showcasing work
register_post_type( 'portfolio', array(
    'labels' => array(
        'name' => 'Portfolio',
        'singular_name' => 'Portfolio Item',
    ),
    'public' => true,
    'has_archive' => true,
    'rewrite' => array( 'slug' => 'portfolio' ),
    'supports' => array( 'title', 'editor', 'thumbnail', 'excerpt' ),
    'menu_icon' => 'dashicons-portfolio',
));

Events Management: Events

// Events with date, location, and registration
register_post_type( 'event', array(
    'labels' => array(
        'name' => 'Events',
        'singular_name' => 'Event',
    ),
    'public' => true,
    'has_archive' => true,
    'supports' => array( 'title', 'editor', 'thumbnail', 'excerpt' ),
    'menu_icon' => 'dashicons-calendar-alt',
));

Real Estate: Properties

// Real estate listings
register_post_type( 'property', array(
    'labels' => array(
        'name' => 'Properties',
        'singular_name' => 'Property',
    ),
    'public' => true,
    'has_archive' => true,
    'supports' => array( 'title', 'editor', 'thumbnail', 'excerpt', 'custom-fields' ),
    'menu_icon' => 'dashicons-building',
));

Key Post Type Parameters

Parameter Description Example
labels Array of labels for the post type name, singular_name, add_new, etc.
public Whether the post type is public true / false
hierarchical Whether posts can have parent/child relationships false (like posts) / true (like pages)
supports Core features the post type supports title, editor, thumbnail, excerpt
has_archive Whether to enable post type archives true / false / 'custom-slug'
rewrite URL rewrite rules array( 'slug' => 'products' )
menu_position Position in admin menu 5, 10, 15, 20, 25...
menu_icon Icon for admin menu dashicons-admin-post, custom URL
show_in_rest Enable Gutenberg editor and REST API true / false
capability_type String used to build capabilities 'post', 'page', or custom

Basic Custom Post Type Example

Complete Book Post Type

// Register Book Custom Post Type
function create_book_post_type() {
    
    $labels = array(
        'name'                  => _x( 'Books', 'Post type general name', 'textdomain' ),
        'singular_name'         => _x( 'Book', 'Post type singular name', 'textdomain' ),
        'menu_name'             => _x( 'Books', 'Admin Menu text', 'textdomain' ),
        'name_admin_bar'        => _x( 'Book', 'Add New on Toolbar', 'textdomain' ),
        'add_new'               => __( 'Add New', 'textdomain' ),
        'add_new_item'          => __( 'Add New Book', 'textdomain' ),
        'new_item'              => __( 'New Book', 'textdomain' ),
        'edit_item'             => __( 'Edit Book', 'textdomain' ),
        'view_item'             => __( 'View Book', 'textdomain' ),
        'all_items'             => __( 'All Books', 'textdomain' ),
        'search_items'          => __( 'Search Books', 'textdomain' ),
        'parent_item_colon'     => __( 'Parent Books:', 'textdomain' ),
        'not_found'             => __( 'No books found.', 'textdomain' ),
        'not_found_in_trash'    => __( 'No books found in Trash.', 'textdomain' ),
        'featured_image'        => _x( 'Book Cover Image', 'Overrides the "Featured Image" phrase', 'textdomain' ),
        'set_featured_image'    => _x( 'Set cover image', 'Overrides the "Set featured image" phrase', 'textdomain' ),
        'remove_featured_image' => _x( 'Remove cover image', 'Overrides the "Remove featured image" phrase', 'textdomain' ),
        'use_featured_image'    => _x( 'Use as cover image', 'Overrides the "Use as featured image" phrase', 'textdomain' ),
        'archives'              => _x( 'Book archives', 'The post type archive label', 'textdomain' ),
        'insert_into_item'      => _x( 'Insert into book', 'Overrides the "Insert into post" phrase', 'textdomain' ),
        'uploaded_to_this_item' => _x( 'Uploaded to this book', 'Overrides the "Uploaded to this post" phrase', 'textdomain' ),
        'filter_items_list'     => _x( 'Filter books list', 'Screen reader text', 'textdomain' ),
        'items_list_navigation' => _x( 'Books list navigation', 'Screen reader text', 'textdomain' ),
        'items_list'            => _x( 'Books list', 'Screen reader text', 'textdomain' ),
    );
    
    $args = array(
        'labels'                => $labels,
        'description'           => __( 'Book custom post type for library website', 'textdomain' ),
        'public'                => true,
        'publicly_queryable'    => true,
        'show_ui'               => true,
        'show_in_menu'          => true,
        'query_var'             => true,
        'rewrite'               => array( 'slug' => 'books' ),
        'capability_type'       => 'post',
        'has_archive'           => true,
        'hierarchical'          => false,
        'menu_position'         => 20,
        'menu_icon'             => 'dashicons-book',
        'supports'              => array( 'title', 'editor', 'author', 'thumbnail', 'excerpt', 'comments', 'revisions', 'custom-fields' ),
        'show_in_rest'          => true, // Enable Gutenberg editor
        'rest_base'             => 'books',
        'rest_controller_class' => 'WP_REST_Posts_Controller',
    );
    
    register_post_type( 'book', $args );
}
add_action( 'init', 'create_book_post_type', 0 );

Planning Your Custom Post Type

Questions to Consider

  1. What content will it hold?
    • What information needs to be stored?
    • What fields are required?
    • Will it need media attachments?
  2. How will it be organized?
    • Does it need categories or tags?
    • Should it be hierarchical?
    • What taxonomies make sense?
  3. Who will manage it?
    • What user roles need access?
    • What capabilities are required?
    • Should it appear in the admin menu?
  4. How will it be displayed?
    • Will it have archive pages?
    • What URL structure is needed?
    • Does it need custom templates?
  5. What features are needed?
    • Comments enabled?
    • Revisions tracking?
    • REST API support?

Best Practices

✅ Do's

  • Choose descriptive, unique post type names
  • Use proper internationalization for labels
  • Register post types on the 'init' action
  • Flush rewrite rules on activation/deactivation
  • Enable REST API support for modern features
  • Consider SEO implications of URL structure
  • Plan for future scaling and requirements

❌ Don'ts

  • Don't use reserved post type names
  • Don't register post types too early or too late
  • Don't forget to handle plugin deactivation
  • Don't hardcode labels without translation
  • Don't create too many post types unnecessarily
  • Don't ignore user capabilities and permissions

Reserved Post Type Names

Avoid using these reserved names for your custom post types:

post page attachment revision nav_menu_item action author order theme

🔐 AJAX Security with Nonces

Protect your AJAX endpoints from attacks

Master WordPress nonces and implement bulletproof AJAX security

Learning Objectives

  • Understand WordPress nonces and their purpose
  • Implement nonces in AJAX requests
  • Protect against CSRF attacks
  • Validate and verify security tokens
  • Handle nonce expiration and renewal
  • Implement additional security layers

Understanding WordPress Nonces

What is a Nonce?

Nonce stands for "Number used ONCE" - but in WordPress, it's a security token valid for 12-24 hours.

📚 Key Facts:

  • User-specific and action-specific
  • Protects against CSRF attacks
  • Changes every 12 hours
  • Valid for up to 24 hours

Complete Secure Implementation

PHP: Server-Side Security

 admin_url('admin-ajax.php'),
        'nonce' => wp_create_nonce('ajax_security_nonce')
    ));
}
add_action('wp_enqueue_scripts', 'enqueue_secure_ajax');

// Secure AJAX handler
function secure_ajax_handler() {
    // 1. Verify nonce
    if (!wp_verify_nonce($_POST['nonce'], 'ajax_security_nonce')) {
        wp_die('Security check failed');
    }
    
    // 2. Check capabilities
    if (!current_user_can('edit_posts')) {
        wp_die('Insufficient permissions');
    }
    
    // 3. Validate input
    $data = sanitize_text_field($_POST['data']);
    
    // 4. Process request
    // Your secure logic here
    
    wp_send_json_success('Processed securely');
}
add_action('wp_ajax_secure_action', 'secure_ajax_handler');
add_action('wp_ajax_nopriv_secure_action', 'secure_ajax_handler');

JavaScript: Client-Side Security

jQuery(document).ready(function($) {
    $('.ajax-button').on('click', function(e) {
        e.preventDefault();
        
        $.ajax({
            url: ajax_object.ajax_url,
            type: 'POST',
            data: {
                action: 'secure_action',
                nonce: ajax_object.nonce,  // Include nonce
                data: 'user_data'
            },
            success: function(response) {
                if (response.success) {
                    console.log('Success:', response.data);
                }
            },
            error: function(xhr) {
                console.error('Security error:', xhr.responseText);
            }
        });
    });
});

Security Threats & Solutions

🚨 CSRF Attack Prevention

❌ Vulnerable Code

// NO nonce verification - DANGEROUS!
function insecure_handler() {
    update_option('critical_setting', $_POST['value']);
    wp_die('Updated');
}

✅ Secure Code

// With nonce verification - SECURE
function secure_handler() {
    check_ajax_referer('my_nonce', 'security');
    update_option('critical_setting', sanitize_text_field($_POST['value']));
    wp_die('Updated securely');
}

Best Practices

✅ Do's

  • Always use wp_create_nonce() and wp_verify_nonce()
  • Create action-specific nonces
  • Implement automatic nonce refresh
  • Combine nonces with capability checks
  • Log failed security checks
  • Use HTTPS to prevent token interception

❌ Don'ts

  • Don't rely on client-side security alone
  • Don't expose nonces in URLs
  • Don't use the same nonce for all actions
  • Don't store nonces in localStorage
  • Don't skip verification for "simple" requests
  • Don't reveal security details in errors

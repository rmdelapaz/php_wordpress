Build a Complete Contact Form Manager Plugin

Apply everything you've learned about admin interfaces, settings, and data storage

Project Overview

Create a professional WordPress plugin called "WP Contact Form Manager" that allows users to create and manage contact forms with a comprehensive admin interface.

This project will test your understanding of:

  • Creating admin menu pages and subpages
  • Implementing the WordPress Settings API
  • Building professional options pages
  • Validating and sanitizing user input
  • Storing and retrieving data from the database
  • Creating custom database tables
  • Using transients for caching
  • Implementing proper security measures

üéØ Core Requirements

1

Plugin Structure

  • Proper plugin header with all required information
  • Organized file structure with separate files for different components
  • Activation, deactivation, and uninstall hooks
  • Support for internationalization (i18n)
  • Follow WordPress coding standards
2

Admin Menu Structure

  • Main menu item with custom icon and position
  • At least 3 submenu pages:
    • All Forms (list of created forms)
    • Add New Form
    • Form Submissions
    • Settings
  • Proper capability checks for all pages
  • Admin notices for user feedback
3

Settings Page

  • Implement using WordPress Settings API
  • Multiple sections with different setting types:
    • General Settings (email notifications, default form settings)
    • Display Settings (form styling options)
    • Advanced Settings (spam protection, data retention)
  • At least 10 different settings fields including:
    • Text inputs
    • Email fields
    • Checkboxes
    • Radio buttons
    • Select dropdowns
    • Textarea
    • Color picker
    • Number input
  • Tabbed interface for settings organization
  • Reset to defaults option
4

Data Validation & Security

  • All user inputs must be validated
  • Proper sanitization before database storage
  • Output escaping for all displayed data
  • Nonce verification for all forms
  • Capability checks for all actions
  • SQL injection prevention using prepared statements
  • XSS attack prevention
5

Database Management

  • Store plugin settings using Options API
  • Create custom table for form submissions
  • Implement data migration system
  • Use transients for caching form data
  • Clean up all data on uninstall
  • Export/Import functionality for settings
6

Form Management Features

  • Create multiple contact forms
  • Edit existing forms
  • Delete forms (with confirmation)
  • List all forms with pagination
  • Shortcode generation for each form
  • Form fields configuration:
    • Name (required)
    • Email (required)
    • Subject
    • Message (required)
    • Custom fields support
7

Submissions Management

  • View all form submissions
  • Filter submissions by form
  • Search functionality
  • Mark as read/unread
  • Delete submissions
  • Export submissions to CSV
  • Email notification on new submission

Starter Code

Main Plugin File Template

<?php
/**
 * Plugin Name: WP Contact Form Manager
 * Plugin URI: https://yourwebsite.com/wp-contact-form-manager
 * Description: A comprehensive contact form management system with advanced admin interface
 * Version: 1.0.0
 * Author: Your Name
 * Author URI: https://yourwebsite.com
 * License: GPL v2 or later
 * Text Domain: wp-contact-form-manager
 * Domain Path: /languages
 */

// Prevent direct access
if ( ! defined( 'ABSPATH' ) ) {
    exit;
}

// Define plugin constants
define( 'WCFM_VERSION', '1.0.0' );
define( 'WCFM_PLUGIN_DIR', plugin_dir_path( __FILE__ ) );
define( 'WCFM_PLUGIN_URL', plugin_dir_url( __FILE__ ) );
define( 'WCFM_PLUGIN_BASENAME', plugin_basename( __FILE__ ) );

// Activation hook
register_activation_hook( __FILE__, 'wcfm_activate' );
function wcfm_activate() {
    require_once WCFM_PLUGIN_DIR . 'includes/class-plugin-activator.php';
    WCFM_Activator::activate();
}

// Deactivation hook
register_deactivation_hook( __FILE__, 'wcfm_deactivate' );
function wcfm_deactivate() {
    require_once WCFM_PLUGIN_DIR . 'includes/class-plugin-deactivator.php';
    WCFM_Deactivator::deactivate();
}

// Load the main plugin class
require_once WCFM_PLUGIN_DIR . 'includes/class-plugin.php';

// Initialize the plugin
function run_wcfm() {
    $plugin = new WCFM_Plugin();
    $plugin->run();
}
run_wcfm();

<?php
class WCFM_Settings {
    
    private $option_name = 'wcfm_settings';
    private $settings_group = 'wcfm_settings_group';
    
    public function __construct() {
        add_action( 'admin_init', array( $this, 'register_settings' ) );
    }
    
    public function register_settings() {
        // Register settings
        register_setting(
            $this->settings_group,
            $this->option_name,
            array( $this, 'sanitize_settings' )
        );
        
        // Add General Settings Section
        add_settings_section(
            'wcfm_general_section',
            __( 'General Settings', 'wp-contact-form-manager' ),
            array( $this, 'general_section_callback' ),
            'wcfm_settings_page'
        );
        
        // Add fields to General Section

            'notification_email',
            __( 'Notification Email', 'wp-contact-form-manager' ),

Database Schema

-- Forms table
CREATE TABLE {prefix}wcfm_forms (
    id bigint(20) NOT NULL AUTO_INCREMENT,
    form_name varchar(255) NOT NULL,
    form_fields longtext,
    settings longtext,
    status varchar(20) DEFAULT 'active',
    created_at datetime DEFAULT CURRENT_TIMESTAMP,
    updated_at datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    KEY status (status)
);

-- Submissions table
CREATE TABLE {prefix}wcfm_submissions (
    id bigint(20) NOT NULL AUTO_INCREMENT,
    form_id bigint(20) NOT NULL,
    user_id bigint(20),
    submission_data longtext,
    user_ip varchar(100),
    user_agent text,
    status varchar(20) DEFAULT 'unread',
    created_at datetime DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    KEY form_id (form_id),
    KEY status (status),
    KEY created_at (created_at)
);

Hints and Tips

Use WordPress's WP_List_Table class for creating professional-looking tables for forms and submissions lists.
Store form field configurations as JSON in the database for flexibility.
Use wp_mail() function for sending email notifications and consider implementing a queue system for better performance.
Implement pagination early to handle large numbers of forms and submissions efficiently.
Make sure to test your plugin with WP_DEBUG enabled to catch any warnings or notices.
Always escape output, even data that comes from your own database. Never trust any data source completely.
